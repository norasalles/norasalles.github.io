Documents Template Related

  • Fixed an issue where when Merging to Doc or Email, Document Templates merge with incorrect formatting including missing line breaks and replacing apostrophes with "&#39;"

    NOTE:

    Document templates (plain text and HTML) saved in CBO 4.09 version are working correctly (newly created or modified). Templates saved in previous CBO versions are not compatible with the new character coding.  To ensure existing templates continue to work, simply edit and resave each one.

 

Commission/GST/HST Related

  • Fixed an issue when creating a new Reservation where the commission rate did not recalculate properly when a multicurrency option is selected and the Vendor is changed.

  • To standardize and increase the transparency of the rules for including and editing Comm GST/HST, we have introduced several changes:

    • For all travel types, if the Reservation is set to International travel regardless to whether Total, Traveler, or Itemized pricing is used, Comm GST/HST is set to zero (0) and not editable and the GST/HST status on the Reservation level is defaulted to Do Not Include GST/HST.

    • When Reservation status is changed from International to Domestic or Transborder or <blank> the GST/HST setting is defaulted to what is set under Agency Settings. (i.e: checkbox Include GST/HST)

    • If Traveler pricing is defined for the Reservation, regardless of what the Domestic/International status and Include GST/HST status is set to, the EDIT button in the Tax section is not presented.  More Taxes and Comm GST/HST fields must be edited under Traveler pricing.

    • When adding Traveler pricing the default value for the GST/HST status is what is set under Reservation for this field.

                                          

      NOTES:

    • After change and saving the GST/HST status under the Reservation, the changed value will not be populated to the existing Traveler pricings.

    • When at least one Traveler pricing is set to Include GST/HST, the pricing at the Reservation level is updated to Include GST/HST even if the default was previously set to Do Not Include GST/HST. In this case, to change the Include GST/HST status at the Reservation level, the user must first change the Include GST/HST at the Traveler pricing level.
