Client Base 4.08.00 Fixes

 

Fixes made between ClientBase Online 4.08.00.121 and 4.09.00.060

 

Live Connect Related

  • Fixed an issue where when during Live Connect import travelers are duplicated upon importing an existing Reservation.

 

  • Fixed an issue where while importing xml reservation through LC provider, the vendor code is not matched with vendor name and the User is prompted that vendor not found although this vendor is connected to used LC provider with assigned vendor code.

 

Query Related

  • Fixed an issue where Profile Manager Query, Res Card Manager Query, Activity Manager Query displayed correct result count but did not display the very top 1st record of the 2nd page in the query result.

 

Navigation Related

  • Fixed an issue where Navigation buttons (IE web browser) for autocomplete list were closed after clicking down or up arrow, or a scroll bar.

 

Credit Card Related

  • Fixed an issue where Credit Card with invalid Expiration Date prevents CBPSync SyncOut

 

Invoice Related

  • Fixed an issue where "Payment due date" is not displayed on the Invoice when reservation has any Itin/Invoice Remarks

 

  • Fixed an issue where the generated Invoice with an empty date "Invoice Payment Due On" defaults to Invoice in Currently Invoice Totals section in the "Payment Due by" date as "December 31, 1979".

 

  • Fixed an issue where Negative Agent Commission value on the invoice generate page is causing that “Generate Invoice”, “Preview” and “Customize Format” buttons are disabled and grayed out (on the top of the page).

 

  • Fixed an issue where "This Invoice Totals": "Base" amount and "GST/HST" amount are incorrect on regular invoice. They are correct on invoice preview.

 

Login page Related

  • Fixed an issue with prompting incorrect error message when expired license access code for CBO.

 

Merge to PNR Related

  • Fixed an issue where when merging to PNR, (Sabre GDS), after clicking Additional Entries button on the PNR Builder page travelers sorting is different, then on the main page, due to that wrong travelers are checked off and when doing preview, entries not for all travelers selected in the main page are merged.

 

ResCard Related

  • Fixed an issue where the order of travelers displayed under created reservation (regardless of the type of reservation) is different than the order of travelers presented under Res Card.

 

Vendor profile Related

  • We added the option to save Vendor Commission rate = 0.00 in the Vendor profile and the setting is honored when creating a new reservation or during PNR import.
