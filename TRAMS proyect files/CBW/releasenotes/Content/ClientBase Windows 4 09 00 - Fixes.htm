ClientBase Windows 4.09.00 Fixes

Changes made between ClientBase Windows 4.08.00.007 and 4.09.00.090

Miscellaneous

  • [EUA Enabled Databases Only] Fixed a messaging issue where, after changing a password, an incorrect second prompt to change the password came up, then the incorrect message "Password Has Been Used Before". 

 

PNR Import Related

- Sabre:

  • Fixed an issue where some PNRs returned a "Not a valid integer value" error and did not import correctly when imported from Sabre Red 360. 
  • Fixed a PNR Import issue for Air bookings where Seat Number and Fare Basis were not imported. Also fixed a related issue where the seat Status defaulted to "Confirmed" rather than "HRS" as in previous versions of ClientBase. 
  • Resolved a PNR Import issue where the Reservation Airline Code for the vendor was not being imported for ticketed Air segments. 
  • Fixed an issue where, for partially ticketed Air bookings, the number of travelers was not imported correctly.

 

- Amadeus:

  • Fixed an issue where, when creating a new Hotel vendor during PNR Import, the Chain Code and Property ID were not saved to the new vendor’s Interface ID field, causing an incorrect prompt to create the vendor each time it was used.
  • Fixed an issue where the same ticket numbers were imported for all Air reservation passengers.
  • Fixed an issue where the Air Reservation number of travelers was set to 1, even when two or more passengers were in the booking.

 

- Worldspan:

  • Fixed an issue when importing Hotel reservations where the Vendor Profile Commission Default and/or Global Default Commission rate was not being applied to the booking.
  • Fixed an issue where importing the same Record Locator into the same Res Card created a new reservation instead of updating an existing record with the same confirmation number.

 

Performance Related

  • Fixed an issue where opening Agent profiles with slow travel history queries took a long time.
  • Corrected an issue where opening group security records, particularly in larger databases, took a long time.
  • Fixed a Merge to PNR performance issue where the PNR Builder window is slow to display if the profile has many associated travelers.

Profile Related

  • Fixed an issue where the Vendor Profile commission default percentage field was sometimes left blank when it should have displayed as 0.00.

Query Related

  • Fixed an issue where the Inventory Query window was hidden behind other ClientBase windows when clicking outside the Inventory window while the query was still running.
