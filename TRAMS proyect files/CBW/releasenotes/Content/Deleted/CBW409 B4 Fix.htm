ClientBase Windows 4.09.00 Fixes

Changes made between ClientBase Windows 4.09.00.084 and 4.09.00.089

Performance Related

  • Fixed an issue where opening Agent profiles with slow travel history queries took a long time.

  • Corrected an issue where opening group security records, particularly in larger databases, took a long time.

  • Fixed a Merge to PNR performance issue where the PNR Builder window is slow to display if the profile has many associated travelers.

 

Changes made between ClientBase Windows 4.09.00.076 and 4.09.00.084

PNR Import Related

  • [Beta Only] [Sabre] Fixed an issue where, for manual service fees and miscellaneous OTH bookings with the "issued through ARC" box unchecked, pricing was populated at the Traveler level rather than at the Reservation level. 

  • [Beta Only] [Sabre] Fixed an issue where, if there was more than one meal served on a flight segment, only one meal was imported. 

  • [Beta Only] [Sabre] Fixed an issue where Service Fee commission was applied to Air bookings as well, resulting in a doubled commission.

 

Security Related

  • [EUA Enabled Only] Fixed a messaging issue where, after changing a password, an incorrect second prompt to change the password came up, then the incorrect message "Password Has Been Used Before"

 

Profile Related

  • Fixed an issue where the Vendor Profile commission default percentage field was sometimes left blank when it should have displayed as 0.00.

 

Changes made between ClientBase Windows 4.09.00.067 and 4.09.00.076

PNR Import Related

  • [Beta Only] [Sabre] Fixed a problem where a "List index out of bounds" error occurred during PNR import if an Other (OTH) segment is the first segment in the PNR.

  • [Beta Only] [Sabre] Fixed an issue where, for partially ticketed Air bookings, the number of travelers was not imported correctly.

  • [Beta Only] [Sabre] Fixed an issue where, when using Sabre Red 360, the number of travelers on Hotel bookings was not imported correctly. 

  • [Beta Only] [Sabre] Fixed an issue where, when using Sabre Red 360, Hotel check-in and check-out times were not imported correctly. 

 

Changes made between ClientBase Windows 4.09.00.057 and 4.09.00.067

PNR Import Related

  • [Amadeus] Fixed an issue where, when creating a new Hotel vendor during PNR Import, the Chain Code and Property ID were not saved to the new vendor’s Interface ID field, causing an incorrect prompt to create the vendor each time it was used.

  • [Amadeus] Fixed an issue where the same ticket numbers were imported for all Air reservation passengers.

 

Changes made between ClientBase Windows 4.08.00.007 and 4.09.00.057

PNR Import Related

  • [Sabre] Fixed an issue where some PNRs returned a "Not a valid integer value" error and did not import correctly when imported from Sabre Red 360. 

  • [Sabre] Fixed a PNR Import issue for Air bookings where Seat Number and Fare Basis were not imported. Also fixed a related issue where the seat Status defaulted to "Confirmed" rather than "HRS" as in previous versions of ClientBase. 

  • [Sabre] Resolved a PNR Import issue where the Reservation Airline Code for the vendor was not being imported for ticketed Air segments. 

  • [Sabre] Fixed an issue where hotel and car segments with more than one passenger in PNR added all travelers to the Service Provider, rather than only the first one.

  • [Amadeus] Fixed an issue where, when creating a new Hotel vendor during PNR Import, the Chain Code and Property ID were not saved to the new vendor’s Interface ID field, causing a prompt to create the vendor each time it was used.

  • [Amadeus] Fixed an issue where the Air Reservation number of travelers was set to 1, even when two or more passengers were in the booking.

  • [Amadeus] Fixed an issue where the same ticket numbers were imported for all Air reservation passengers.

  • [Worldspan] Fixed an issue where the Vendor Profile Commission Default and/or Global Default Commission Rate was not populated for Hotel reservations without commission in the PNR booking.

  • [Worldspan] Fixed an issue where importing the same Record Locator into the same Res Card created a new reservation instead of updating an existing record with the same confirmation number.

  • [All GDS] Fixed an issue where some segment statuses imported into Reservation Agent Remarks with only a short status description, e.g. "Status: Confirmed." We now always import the 2-letter status code and the long status description, formatted like this: "Status: <2-letter Status Code> <Long Status Description>."

  • [All GDS] Fixed an issue where Tour and Cruise segments with a linked accounting line were reading the vendor code from the accounting line instead of from the segment.

 

Merge to PNR Related

  • Corrected the Traveler Miscellaneous merge field names Emergency Phone Country Code, Emergency Phone Area Code, and Emergency Phone Prefix/Suffix to fields Emergency Contact Phone Country Code,  Emergency Contact Phone Area Code, and  Emergency Contact Phone Prefix/Suffix, respectively.

 

Performance Related

  • Fixed an issue where the Inventory Query window was hidden behind other ClientBase windows when clicking outside the application before the query completes.
