Specification Table

Element Name Data Type (Length) Example Required Travel Categories Notes
VENDORCODE Char(10) ABC Y All Code for the vendor.We can assign one if the vendor does not have one that they use. This is used to find a match on the vendor profile in CBPlus.For backward compatibility. Vendor information is on the Vendor node.
VENDORNAME Char(80) ABC Vacations Y All Name of the vendor or the booking site.
RESTYPE Integer 0 = Air 1 = Hotel 2 = Car 3 = Cruise 4 = Insurance 5 = Miscellaneous 6 = Rail 7 = Tour 8 = Service Fee 9 = Transportation Y All Reservation type.
RESDATE Date MMDDYYYY Y All Date of reservation.
CONFIRMATIONNUMBER Char(20)   N All Reference number assigned to the reservation.
RECORDLOCATOR Char(20)   N All CRS record locator number.
PROMOID Char(20)   N All  
SOURCEOFBOOKING Char(20)   N All Name of the site where the reservation is made.This is useful if there is more than one site that the agent can book with a particular vendor.
BOOKINGSTATUS Char(20) Confirmed Quoted Cancelled N All The difference between this and the STATUS field is that this affects the Reservation Totals (Reservation, Invoiced, Non-Invoiced Amounts). This only has 3 valid values. Note:If you use Quote, ClientBase will not allow an Invoice to be priced, but will show the amounts in an Itinerary.
STATUS Char(20) Confirmed, Quoted, Cancelled, On Hold, On Request, etc. N All Status of the reservation. This is where the status of the booking within the supplier system should be stored. Use any Item description you wish:
TRAVELDESTTYPE Char(1) D = Domestic I = International T = Transborder N All Travel Destination type.
GROUPID Char(128)   N All Group Contract ID.
RATECODE Char(128)   N All  
INVOICEREMARKS Char(4000)   N All Remarks that show when the reservation is invoiced in CBPlus. Use “|” to separate remark lines.
AGENTREMARKS Char(4000)   N All Remarks in this section do not print in invoice. Use “|” to separate remark lines- Good place to put Marks if included in gross amount as is the case in Itemization.This section will not print on an invoice.
TOTALBASE Integer 15000 (150.00) Implied decimal point location: 2 places to the right N All Base amount of the reservation.
TOTALTAX Integer 1500 (15.00) N All Tax amount of the reservation.
TOTALTAXPERCENT Integer 1000 (10.00) N All Tax percent relative to the base amount.
TOTALCOMMISSION Integer 1500 (15.00) N All Commission amount of the reservation.
TOTALCOMMPERCENT Integer 1000 (10.00) N All Commission percent relative to the base amount.
COMMGST Integer 150 (1.50) N All GST/VAT on commission.
CLIENTGST Integer 1500 (15.00) N All GST/VAT.
TAX1AMT Integer   N All  
TAX1RATE Integer   N All  
TAX2AMT Integer   N All  
TAX2RATE Integer   N All  
TAX3AMT Integer   N All  
TAX3RATE Integer   N All  
TAX4AMT Integer   N All  
TAX4RATE Integer   N All  
HIGHFARE Integer 1500 (15.00) N All  
LOWFARE Integer 1500 (15.00) N All  
FARECODE Char(6)   N All Explanation Code for High and Low Fare.
ADJUSTMENTTYPE Integer 1 = Markup 2 = Discount N All Does not work with Itemized Pricing.
ADJUSTMENTAMT Integer 5000 (50.00) N All  
ISSUEDTHROUGHARC Char(1) Y/N N All Issued through ARC/BSP.
STARTDATE Date MMDDYYYY N All Date when the trip starts. (Calculated from earliest Service Provider date).
ENDDATE Date MMDDYYYY N All Date when the trip ends. (Calculated from latest Service Provider date).
DURATION Integer   N All Duration of the trip (Number of nights/days).
NOOFUNITS Integer   N All Used in the following Travel Categories: Hotel: No. of Rooms Car: No. of Cars Cruise: No. of Cabins Insurance: No. of Policies Miscellaneous, Service Fee, Transportation: No. of Units.
NOOFPAX Integer   N All The number of passengers (including infant or lap child) for all Travel Categories.
CHARGEDAS Integer 1 = per duration; 2 = per unit; 3 = per passenger 0 = if amounts are already for the Reservation totals. N All This is dependent on the pricing fields above (TOTALBASE, TOTALTAX, TOTALCOMMISSION). If the prices are the total for the whole reservation, then assign 0 on the CHARGEAS field. If the prices refer to per passenger, then this will be 3. 1 if it's per night/day or 2 if its per policy etc. In the case of Tour reservations, this is usually set to 0.
DEPOSITDUEDATE Date MMDDYYYY N All Date the deposit is due or date payment was made.
FINALPYMNTDUEDATE Date MMDDYYYY N All Date the final payment is due.
AMOUNTPAID Integer 15000 (150.00) N All Amount Paid.
FORMOFPAYMENT Integer 1 = Cash 2 = Check 3 = Credit Card 4 = Other 5 = CC Merchant 6 = EFT 7 = N/A N All  
PAYMENTNO Char(20) 37681488555555 N All Check or Credit Card #.
ALLOWMULTICURRENCY Char(1) Y=Yes N All Required if Client and/or Vendor Currency is used.
CLIENTCURRENCY Char(3) CAD (Canada) N All Currency used by Client.
CLIENTCONVRATE Integer 150 (1.5) N All From Agency Currency.
CLIENTBASEAMT Integer 15000 (150.00) N All  
CLIENTTAXAMT Integer 1500 (15.00) N All  
CLIENTTAXRATE Integer 1000 (10.00) N All  
CLIENTCOMMAMT Integer 1500 (15.00) N All  
CLIENTCOMMRATE Integer 1000 (10.00) N All  
VENDORCURRENCY Char(3) CAD (Canada) N All Currency used by Vendor.
AGENCYCONVRATE Integer 150 (1.5)     From Vendor Currency.
VENDORBASEAMT Integer 15000 (150.00) N All  
VENDORTAXAMT Integer 1500 (15.00) N All  
VENDORTAXRATE Integer 1000 (10.00) N All  
VENDORCOMMAMT Integer 1500 (15.00) N All  
VENDORCOMMRATE Integer 1000 (10.00%) N All  
CONVERSIONRATEDIVISOR Integer 1000 or 100 N All 1000 is for 4 decimal places; 100 for 2.
