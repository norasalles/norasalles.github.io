Application Screen:

 

 

Specification Table:

 

Element Name Data Type (Length) Example Required Travel Categories Notes
LAST NAME Char (30)   Y All Passenger last name.
FIRSTNAME Char (30)   Y All Passenger first name.
MIDDLENAME Char (30)   N All Passenger middle name.
PASSENGERTYPE Char (1)

A = Adult

C = Child

I = Infant

 Y All Type of passenger. Lap child are considered infant. Defaults to Adult.
PRIMARYPASSENGER Char (1) Y/N N All If no node or not set, the first passenger will be set to the primary.

<RESCARD>
    <!-- other res card elements here -->

        <PASSENGER>
            <FIRSTNAME>Timothy</FIRSTNAME>

            <PRIMARYPASSENGER>N</PRIMARYPASSENGER>
        </PASSENGER>
        <!-- PASSENGER elements cont. -->
    </PASSENGERLIST>
