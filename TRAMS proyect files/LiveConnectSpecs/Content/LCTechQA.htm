Live Connect Tech Q & A

What does Live Connect use to brows to a booking site?

 

Live Connect uses shdocvw Internet Explorer object (WebBrowser control) embedded on a form. The object is an instance of IE and uses the same IE settings set under Internet Options. It emulates on the same IE version when registry entries for ClientBase is updated.

 

How does Live Connect send information to a booking site?

 

Live Connect navigates to the provider URL and sends an HTTP POST with the login, profile, traveler and reservation (for insurance sites) information.

 

What indicates that the user wants to retrieve an existing reservation?

 

The user is prompted in ClientBase with the confirmation number. When the ConfirmationNo input has a value on the post, the ideal result is for the booking site to land them on the confirmation page of the reservation.

 

How does Live Connect get the reservation details from a booking site?

 

There is an Import Reservation button on the form that houses the WebBrowser control. When clicked, Live Connect reads the source of the page and looks for the XML (<RESCARD> tag) on the body of the confirmation page. The XML is just a hidden comment. Live Connect also checks the source of the frames on the page.
