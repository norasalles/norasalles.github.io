Live Connect Features

1. Login passes automatically

 

Travel agents can store their user name and passwords for each participating supplier’s booking engine inside of their ClientBase profile. If you include the ability to receive their login credentials via the HTTP POST the agent will automatically be logged in when connecting to your booking site from ClientBase.

 

2. Client data passes automatically

 

Your booking site should read the customer profile data (e.g. Legal name, address, passport, birth date, frequent traveler numbers, credit cards, etc.) that the agent selects when beginning a Live Connect session for a new booking. This information is sent via the HTTP POST and then used to populate the appropriate fields of your website throughout the booking process.

 

3. Reservation data from your booking engine's confirmation page imports for new bookings

 

Once an agent completes a new reservation via a Live Connect session they should be able to import the reservation details back into their ClientBase ResCard from your confirmation page.

There is an Import Reservation button on the form that houses the WebBrowser control. When the agent clicks the button, Live Connect reads the HTML source of the page and looks for the XML (<RESCARD> tag) within the body tags of the confirmation page. The XML should be hidden on the page as a comment or via another HTML method. Live Connect also checks the source of each frame on the page if frames are used.

 

4. Import Existing Reservation (in 2 possible ways)

 

a. Allow user to browse to existing reservations via the website and import.

You will allow an agent to use the normal website navigation to access and display a previously booked reservation which may or may not have been created via Live Connect (agent may have booked over the phone or via your website directly) and import the XML reservation details from the confirmation page back into their ClientBase ResCard.

 

b. Site retrieves confirmation number from POST and immediately displays reservation details for import. (Preferred Method)

 

As with a. but instead of using website navigation, you retrieve the confirmation number directly from the HTTP POST that is sent from ClientBase and the user is taken directly to the booking’s confirmation page for import.

5. Retrieve previously imported reservation for viewing or editing and imports updated reservation details

 

Under the same options as a. and b. for #4, this allows an agent to access an existing booking via Live Connect consecutive times particularly to import updated reservation details (e.g. added transfers, updated cost/new commission, added insurance, room or stateroom upgrades).
