What happens during a Profile Merge?

When a Sabre Profiles Profile is merged into a matching ClientBase Profile, if the corresponding ClientBase field is empty and the Sabre Profiles field contains data, we populate the ClientBase field with the Sabre Profiles data. Similarly, if the corresponding Sabre Profiles field is empty and the ClientBase field contains data, the Sabre Profiles field is populated with the ClientBase data. If both contain data, then whichever record was updated more recently will update the other side.

This rule applies when there is only one record allowed for the type, for example, Known Traveler Number, Name fields, and Birth Date fields. When more than one record of a given type is allowed, for example, phone numbers, payment cards, and passports, we attempt to match up those records and update them, but if we cannot, we will insert as an additional record.
