Wrapper Profile Creation

If a Sabre Profiles Traveler Profile does not have any valid associations to any other ClientBase Profiles, a new Leisure “wrapper” ClientBase Profile is created. Fields imported are First Name, Last Name, Middle Name, and Courtesy Title. The Sabre Profiles Traveler Profile is linked into this Leisure “wrapper” ClientBase Profile. (If desired, this new Profile can be changed from Leisure to Corporate by using the Type drop-down in the General Info tab.)

These wrapper profiles do not sync to Sabre Profiles. They exist solely to contain standalone Sabre Profiles Travelers.

When will a ClientBase wrapper profile be created for a Sabre Traveler, and when will it not? Let’s look at the possible scenarios.

Sabre Profiles

ClientBase

A Traveler Profile is associated with one or more Sabre Profiles Corporate or Group profiles

No wrapper profile is created; upon sync, the Traveler will be linked to the same profile or profiles in ClientBase, preserving the association.

A Traveler Profile with no associated profiles existing in the ClientBase database, but it has a AN or DK Number that exists in one or more ClientBase Leisure or Corporate profiles

No wrapper profile is created; the traveler will be linked to the matching profile or profiles.

A Traveler Profile with no AN or DK number and no associated profiles in the ClientBase database

A Leisure wrapper Profile is created. The Interface ID is populated based on the format for generating Interface ID under Default Values for New Profiles if specified, or left blank if no such default format is established

A Traveler Profile with no associated profiles in the database and has a AN/DK Number that exists in zero or more than one ClientBase Leisure or Corporate profiles

 

A Leisure wrapper profile will be created. The Interface ID will be populated with first AN/DK Number Any other AN/DK numbers will be discarded.
