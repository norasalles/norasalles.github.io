Corporate and Group Profiles Matching Logic

Section 4

If no records are found via Sections 1, 2, or 3, OR there are still multiple AN and/or DK matches that haven't been resolved...

Insert a new Profile:

  1. Of type Corporate if SP Profile Type is CRP.
  2. Of type Leisure if SP Profile Type is GRP.
 If the SP Profile has AN Numbers...

Set PROFILE.INTERFACEID to the first available value (after Section 2 is performed).

 If the SP Profile does not have AN Numbers and does have DK Numbers...

Set PROFILE.INTERFACEID to the first available value (after Section 3 is performed).

 If the SP Profile does not have an AN or a DK...

Leave PROFILE.INTERFACEID blank.

 If the SP Profile has both AN and DK numbers...
  1. Set PROFILE.INTERFACEID to the first available AN or DK value (AN takes priority).
  2. Treat rest of the sync in of the SP Profile as an update.
