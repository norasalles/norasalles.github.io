Corporate and Group Profiles Matching Logic

Section 13

If the Sabre Traveler has no valid Profile associations...

Look for a profile having a Traveler with a matching PASSENGER.SPUNIQUEID.

 If one match found...

Update that Traveler with the SP Traveler data.

 If more than one match found...

Log an error. Do not attempt any further action.

 If no matching Traveler is found via PASSENGER.SPUNIQUEID...

Look for a Corporate or Leisure profile with an Interface ID matching the Traveler's AN/DK.

 If no Corporate or Leisure profile with an Interface ID matching the Traveler's AN/DK is found...

Create a wrapper profile of type Corporate.

  1. Create a new Traveler within the new wrapper profile using the Traveler data.
  2. Populate the wrapper profile's Interface ID with the first AN or DK of the SP Traveler record.
 If one Corporate or Leisure profile with an Interface ID matching the Traveler's AN/DK is found...
  1. Look at the Travelers within that Profile.
  2. Look for a match by Known Traveler Number.
 If one match is found by KTN...

Update the CB Traveler with the SP Traveler data.

 If no match is found by KTN, or if there are multiple matches by KTN...

Look for a match by Passport Number + Issuing Country.

  • Note: Both fields must match in order for this to be considered a match.
  • The matching passport may be in Passport #1 or Passport #2.
 If one match is found by Passport number + Issuing Country...

Update the CB Traveler with the SP Traveler data.

 If no match is found by Passport number + Issuing Country, or if there are multiple matches by Passport number + Issuing Country...

Look for a match by First Name + Middle Name + Last Name + Birth Date.

  • All four fields must match in order for this to be considered a match.
  • Middle name can be considered a match if CB, SP, or both have a blank field.
  • Birth Date can be considered a match if both sides are blank; if there is a value present both sides have the same value.
 If a match is found by First Name + Middle Name + Last Name + Birth Date...

Update the CB Traveler with the SP Traveler data.

 If no match is found by First Name + Middle Name + Last Name + Birth Date, or if there are multiple matches by First Name + Middle Name + Last Name + Birth Date...

Insert as a new Traveler within the Profile.

 If more than one Corporate or Leisure profile with an Interface ID matching the Traveler's AN/DK is found...
  1. Look at the Travelers within each Profile, one by one.
  2. Look for a match by Known Traveler Number.
 If one match is found by KTN...

Update the CB Traveler with the SP Traveler data.

 If no match is found by KTN, or if there are multiple matches by KTN...

Look for a match by Passport Number + Issuing Country.

  • Both fields must match in order for this to be considered a match.
  • The matching passport may be in Passport #1 or Passport #2.
 If one match is found by Passport number + Issuing Country...

Update the CB Traveler with the SP Traveler data.

 If no match is found by Passport number + Issuing Country, or if there are multiple matches by Passport number + Issuing Country...

Look for a match by First Name + Middle Name + Last Name + Birth Date.

  • All four fields must match in order for this to be considered a match.
  • Middle name can be considered a match if CB, SP, or both have a blank field.
  • Birth Date can be considered a match if both sides are blank; if there is a value present both sides must have the same value.
 If one match is found by First Name + Middle Name Last Name + Birth Date...

Update the CB Traveler with the SP Traveler data.

 If no match is found by First Name + Middle Name + Last Name + Birth Date, or if there are multiple matches by First Name + Middle Name + Last Name + Birth Date...

Check the next profile until you reach the end of the list.

 If no match found in any of these profiles, or if there are still multiple matches...

A new wrapper profile and new passenger (traveler) record shall be created in CB.
