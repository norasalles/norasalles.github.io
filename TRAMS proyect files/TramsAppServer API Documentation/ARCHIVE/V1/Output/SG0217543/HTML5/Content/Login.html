Login

Login validates the username and password for the specified Alias/PCC and returns a SessionID token.

Usage: <ServerURL>/login

Login parameters need to be specified in the content portion of the request:

  • Request content:
{
  "username":"SYSDBA", 
  "password":"masterkey", 
  "alias":"TRAMS", 
  "appType":"0",
  "accessKey":""
}
  • Username, Password, and Alias are required parameters.
  • Alias should correspond to a database alias that's been configured via the TramsAppServerSettings tool.
  • AppType is optional. Valid AppType values are 0 for None, 1 for TBO, and 2000 for CB. When specifying an AppType, the logged in user will have the same permissions they would when using the corresponding application.
  • AccessKey is optional. AccessKey is used to specify a key that will unlock certain API capabilities. It's not enabled at this time, but may be enabled before general release.

 

Response: Login returns the standard response with the addition of the SessionID token if the login was successful (resultcode=0):

{
    "apiVersion": "4.05",
    "result": {
        "resultcode": 0,
        "resulttext": "",
        "SessionID": "<SessionID token>",
        "APICapabilities": [ ]
    }
}
