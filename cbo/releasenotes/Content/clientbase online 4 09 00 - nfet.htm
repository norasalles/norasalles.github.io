ClientBase Online 4.09.00 - New Features and Enhancements

Released Sep, 2020

 

ClientBase Online 4.09.00 New Features and Enhancements

Enhancements made between ClientBase Online 4.08.00.121 and 4.09.00.107

Profile Related

  • You can now select from two additional gender types, X (Unspecified) and U (undisclosed) from the Gender dropdown list for Travelers and Contacts.
  • To streamline the process of creating a new profile, we added the Sabre PCC field on the Quick Entry form for new Agent profiles.  The assigned Sabre PCC will populate on each new profile created by the user with the assigned agent. (ClientBase sync with Sabre Profiles activated agencies only)

 

Usability/Accessibility Related

  • Updated the message presented on the Temporary Password page and the Login Security Settings page when password requirements are not met in the case of medium and high strength passwords.
  • ClientBase Online now provides more informative messages when your agency database is currently disabled or disconnected.
  • For increased intuitiveness with the Help menu (?), we updated the Contents menu label to ClientBase Help and added a new ClientBase Tutorial link which leads to Trams and ClientBase Learning Center.
  • To improve password creation usability, we added password length validation so that it is not possible to save a password longer than 32 characters.

 

Merge to PNR Related: All GDS

  • The new Gender types X (Unspecified) and U (undisclosed) will merge to PNR using the existing «Gender» merge field and any rules you have already created using this merge field.
  • To enable you to create a Merge to PNR gender rule for American Airlines, which requires the gender for infants to use only the standard gender format "M", "F", "X" or "U" not including post-fix "I", a new merge field, «Gender (AA)», has been added to the Traveler Miscellaneous field group.
  • To help you comply with IATA rule 830d, which requires e-mail address and phone numbers to be in specific formats, new PNR merge fields have been added.

Traveler Contact Email and Phone:

  • A new E-mail merge field, «Traveler Contact Primary Email», has been added to the Traveler Miscellaneous field group. Each GDS requires a slightly different format for the Traveler’s contact email address, and this merge field will automatically merge the correct format for the selected GDS.  It is important to note that this merge field transmits the Primary email address assigned to the specific traveler. Profile level email addresses cannot be merged when using this field. 
  • A new Phone merge field, «Traveler Contact Primary Phone», has been added to the Traveler Miscellaneous field group. The Traveler Contact Primary Phone merge field merges the Primary phone number of the traveler. Profile level phone numbers cannot be merged when using this field.
  • Important note on Traveler Contact phone and e-mail: A 2-letter code to identify your client’s preferred language is optional. Currently, ClientBase does not have a profile field to identify a language preference.  If passing this code is necessary, you will need to use the Merge to PNR preview option to add the code to the format before sending it to your PNR.

 

Emergency Contact Email and Phone:

  • To allow you to create Merge to PNR rules that will merge the Emergency Contact e-mail address in the airline-required format (replacing the @ with a //), a new merge field, «Emergency Contact E-mail - //», has been added to the Traveler Miscellaneous field group.
  • The former «Emergency Contact E-mail» merge field is now called «Emergency Contact E-mail - @», and will merge the email address with the normal @ format. This field can be used for email templates and anywhere else you would use the usual @ formatted email address.
  • Because the Emergency Contact phone number must be merged to PNR without a dash between the phone prefix and suffix, a new merge field, «Emergency Contact Phone Prefix/Suffix Without Dash», has been added to the Traveler Miscellaneous field group. Please note: ClientBase will read the Emergency Contact e-mail address and the Emergency Contact phone identified in the Advanced Traveler Info tab of the corresponding Traveler record.
  • Important note on Emergency Contact phone and e-mail: Since we currently do not have a citizenship field for emergency contacts, we recommend using the Citizenship merge field for the traveler as the closest alternative.
  • Please see the table at the end of this document for specific formats per GDS.

 

PNR Import Related: All GDS

  • Updated how segment status imports into Reservation Agent Remarks with the 2-letter status code and the long status description, formatted like this: "Status: <2-letter Status Code> <Long Status Description>." 

 

PNR Import Related: Sabre Red 360

  • For more efficient verification and troubleshooting, we added the ability to save a PNR import data to file. When a user goes from ResCard to the PNR import page and selects Sabre GDS, a checkbox is presented to give the user the possibility to import and save PNR to file.
  • Added the ability to import a previously saved PNR into a ResCard. Simply select Sabre GDS from the dropdown, enter an asterisk symbol “*” in the Record Locator field, and indicate the path location where the file was saved.
  • When the Service Provider name is not present in the PNR, then the Service Provider name, address, phone, and fax are populated using the Reservation Vendor’s information.
  • Updated the PNR Import process to use Sabre’s new GetReservation API, which replaces the obsolete GetItinerary API. Most reservations will import as they always have; however, we were able to make use of some new capabilities in GetReservation to improve the details we import. Here is a summary of changes by Travel Category:

 

Transportation:

  • BUS bookings used to import as Rail and will now import as Transportation.
  • LAN/ATX bookings previously imported as Transportation and will still do so.
  • LIMO/SUR bookings will not be imported.

 

Car:

  • Car segments now import into reservations with only the traveler name the car reservation was confirmed under.
  • For Car bookings, we now calculate the base fare by subtracting the Mandatory Charges (taxes and fees) from the Approximate Total Fare, if present. For passive segments, that do not have an Approximate Total Fare, we import the Vehicle Charge Amount as the base fare. 
  • Car commissions are now imported into ClientBase Online reservations. A commission in a car segment can be expressed as an amount or a percentage and will populate the commission amount or rate fields accordingly. If the commission value is not defined for the reservation in the PNR, the ClientBase Online Vendor Commission Default Rate is used to calculate it; if no Vendor Default Rate is available, the ClientBase Online Global Commission Default rate is used.

Note: While we import international car reservations with the currency amount they were booked in, the currency type in CBO will still reflect the agency's default currency setting. The user will need to manually convert the correct cost into their agency currency or select the proper currency code if the agency is using the foreign currency feature. 

 

Insurance:Pricing and Commission are now imported from the accounting line.

Cruise:

  • The accounting line is ignored because cruises are not normally paid in full at the time of booking.
  • ClientBase Online now reads and imports the following cruise segment data.

Tour: The base tour amount is imported from the tour segment in the PNR. The accounting line is ignored because tours are not normally paid in full at the time of booking.

Rail: The number of travelers on the reservation is now accurate. Pricing and commission are now read from the accounting line if present. Ticket numbers are also imported if present.

Hotel:

  • Hotel pricing is now imported and calculated per room, rather than per person or per night. This resolves issues where amounts were calculated incorrectly due to different room rates charged on different nights.
  • For Hotel bookings, the Sabre Rate Description now populates the Room Description field of the Service Provider. 
  • Hotel segments now import into Reservations with only the Traveler name the hotel reservation was confirmed under, rather than listing all the Res Card travelers.
  • The Hotel Total Tax is now imported as part of the hotel pricing.  When a hotel booking is imported, if it contains the Total Tax, we populate the Reservation's Tax field with that amount. If no Total Tax is provided, the Reservation's Tax field is left blank. The Reservation’s Base Fare is calculated by deducting the Total Tax and Surcharge amounts from the Approximate Total cost of the hotel stay. Note: Sabre is still working on making the Total Tax amounts for CSL hotel bookings available, so be sure to double-check the amounts as they may be missing or inaccurate until this work is completed.
  • Hotel commissions are now imported into Reservations. A commission in a hotel segment can be expressed as an amount or a percentage; we perform necessary calculations and populate the commission amount and rate fields accordingly. For commissionable bookings with no commission in the PNR, the Vendor Commission Default Rate is used to calculate it; if no Vendor Default Rate is available, the Global Commission Default rate is used. No commission is imported for non-commissionable bookings (indicated by the code NC).

Note: While we import international hotel reservations with the currency amount they were booked in, the currency type in ClientBase Online will still reflect the agency's default currency setting. The user will need to manually convert the cost into their agency currency or select the proper currency code if the agency is using the foreign currency feature.

  • Added support for the new hotel formats for Content Services for Lodging (CSL) bookings created via aggregators (Booking.com, BedsOnline.com, and Expedia Partner Solutions).  In order to correctly import bookings using the aggregator as the Vendor, you will need to create a Vendor profile for each one using the chain code and property code as the Interface ID. (The correct Interface ID for Booking.com is 7B0, for BedsOnline it is 7H0, and for Expedia it is 7E0.) The hotel property details will be found under the Service Provider.

Other: OTH Segments are imported only if there is an associated accounting line.  This is to prevent importing informational OTH segments, such as retention lines used to keep the PNR active in Sabre longer.

Service Fee: Service fees created using Sabre Merchant Services now import as a Service Fee rather than a Miscellaneous reservation in ClientBase when using "FEE" as the charge descriptor code in your accounting line.  The charge descriptor code can be updated in your Merchant Profile settings. ARC/BSP service fees import as they always have.

  • For Service Fee and Other bookings, the type field indicators PER, ALL, and ONE are now read and calculated accordingly:

  • PER: This fee is charged for each traveler and will be multiplied by the number of travelers
  • ALL: One fee for all the travelers, no matter how many
  • ONE: This fee applies to one specific traveler

 

 

MCO: For miscellaneous charge order segments pricing and commission will be read if associated with an accounting line.  When using MCO segments to identify airline service fees, these will be imported as one Miscellaneous reservation with no pricing for the actual MCO segment and a separate reservation with pricing and commission for each automated Service Fee accounting line present in the PNR.

 

Air:

  • The Class of Service for air segments is now obtained through a new web call that pulls the latest information filed by the airline through ATPCO (Airline Tariff Publishing Company) daily.  This will ensure the correct booking class is reflected in your ClientBase trip proposals and invoices. There is space for 20 characters in ClientBase, so any longer class of service names will be truncated. 
  • Enhanced PNR Import to correctly interpret Passenger Type codes. Previously, any code that was not recognized resulted in the passenger being marked as an Infant. Now, types INF and INS are imported as Infant; types CNN, C03, C04, C05, C06, C07, C08, C09, C10, and C11 are imported as Child; and ADT, NEG, and all other types are imported as Adult
  • Air bookings imported from PNR will no longer pull commission from the Global Commission Defaults. If a commission is defined in the PNR booking, then commission will be imported from the accounting line. If there is no commission [commission is blank or not present], then ClientBase will check the Vendor profile for a Vendor-level commission default, and if found, will populate the reservation commission using it. If there is no vendor-level commission default, then the commission amount is zero. 
  • Exchange tickets now update the existing reservation with the add collect amount rather than importing as a new booking. Note: Be sure to remove any original stored price quotes (PQ) from the PNR to import the new and correct Fare Basis code.
  • For fares involving multiple airlines validated on the international carrier, the ticketing carrier will import as the Reservation vendor in ClientBase Online.  All individual flight segments will import as Service Providers.
  • When importing a Codeshare flight, the operating carrier will be identified in the description field under the Service Provider’s Flight Details tab.
  • Direct Flights, with a stop but no change of aircraft, will now import showing each leg of the flight separately.
  • When issuing multiple tickets using segment select in the same PNR, each ticketed airline is imported as a separate Reservation and the associated flight segments are imported as Service Providers.
  • When a booking with a price quote exists in a PNR along with segments that have accounting lines, the price quoted reservation is now also imported with the quoted pricing.
