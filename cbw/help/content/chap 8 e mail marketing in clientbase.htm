Chapter 8: E-mail Marketing in ClientBase

E-mail Marketing in ClientBase

This section demonstrates how to setup and use Merge to E-mail in ClientBase to market to clients. It further outlines how to track mailings, and run follow-up reports.

Introduction

There are two e-mail options available for using ClientBase Merge to E-mail feature: MAPI (Messaging Application Protocol Interface) and SMTP (Simple Mail Transfer Protocol). MAPI requires use with compatible e-mail programs such as Microsoft Outlook, Microsoft Express, Netscape and Eudora. E-mails sent with this option can be formatted in one of two modes - Plain Text or Document Template. Setting the mode to document template allows you to insert merge fields into the body of a message, or select from a list of global default document templates.

The SMTP option offers 2 benefits: It does not require a MAPI compatible e-mail program for the Merge to E-mail feature to work, and with this option, you can use extensive text formatting, even using HTML, as well as using document templates within e-mail messages.

The option for using either MAPI or SMTP as your method for sending e-mails is located within each user login. Upon installation of ClientBase (or update), all user logins continue to default to MAPI as the method, and each user login record needs to be modified to change the option to SMTP, if desired, and to capture the necessary e-mail settings. Initial setup also includes adjusting or checking e-mail settings at the workstation level because these override the user level defaults.

Users can create e-mail signatures for use with the ClientBase Merge to E-mail feature. A global set of e-mail signatures can be created using formatted text, user login and agent merge fields and graphics. Each user login can link to one of these global e-mail signatures to set as that user’s default signature. Upon launching Merge to E-mail, the system automatically launches the current user’s default signature or an e-mail signature can be selected at that time.

Set Up E-mail Signatures

This option only works if e-mails are set up as SMTP. Go to Global Defaults|General Setup|E-mail Signatures and click Add.

Enter a brief description for the signature. Next, create an e-mail signature using formatted text and merge fields. If desired, use the blue and yellow button (with a picture of a person) to insert a graphic file. Click OK to save and repeat these steps to create multiple e-mail signatures.

Upon creating your global e-mail signatures, have each user go to Utilities|My Login and click the E-mail settings tab to select a default e-mail signature for each User.

Set Up for MAPI E-mail

Some sample MAPI compatible e-mail programs include Microsoft Outlook, Microsoft Express, Netscape, and Eudora. To learn if your e-mail program is MAPI compatible, we recommend you visit the website of your e-mail provider and search for information on MAPI compatibility. When you install the ClientBase program, MAPI is the option which is defaulted in the user logins.

Using Merge to E-mail Configured for MAPI

The first step in using the Merge to E-mail feature is to use the Profile Manager to find the profile(s) to Merge to E-mail by using Level 1, 2, or 3 Query. E-mail addresses are kept in the communication entry table found in the general information tab of all types of profiles.

When merging information into an e-mail, merge the information to e-mail from the results screen:

1) Click the toolbar e-mail icon or select the menu item Merge To|E-mail, or;

2) Double-click a profile to enter the profile folder. From within the profile, click the toolbar E-mail icon.

Sort by: Choose how e-mails are sorted. Choices include Name, First Name, Last Name, and E-mail (default).

Include Records: Select Current Results in Profile Manager (one e-mail is created for all profiles included in the results screen); or Selected Results in Profile Manager (only those profiles that have been selected from results screen by holding down CTRL key and selecting and/or by right-mouse clicking results screen to Invert All, Select All or Unselect All).

Look for E-mails In: Profile (If none, then Primary Traveler) - This option sends to the profile-level e-mail address, if there is one. If there isn't, then it will use the e-mail address of the primary traveler.

Traveler (If none, then Profile) - If no traveler has an e-mail address that meets the query criteria, then e-mail is sent to the profile address or addresses that meet the criteria.

Traveler Only - Looks only for traveler e-mail addresses based on matching query criteria. This option merges to all traveler e-mail addresses that meet the criteria. If there isn't one, then nothing is sent.

All (Profile and Traveler) - this option sends to all addresses in the profile that meet the query criteria.

Include Only E-mails With Marketing Permission Checked: By default this is unchecked to include only e-mails with marketing permission.

Include Only Primary E-mail: this box is checked by default. For any of the above options, if this box is checked, only the Primary E-mail address (either traveler, profile, or both, depending on the option chosen) will be sent to. If it's unchecked, then all e-mails, primary or non-primary, that meet the criteria will be used.

Use BCC Method of Sending E-mail: For users who want to send one e-mail with all other recipients as Blind Carbon Copies. This avoids having to delete many e-mails out of your Sent Box for MAPI users. It is a good idea to input your own e-mail address in the To field as anti-Spam programs need to see at least one e-mail address. Suggestion: Don’t use this option.

Set Up for SMTP E-mail

If your agency would like to send formatted text or HTML e-mails, your mail server and ClientBase e-mail settings must be configured for SMTP.

Finding Mail Server’s SMTP Settings

SMTP is a standard protocol (language) used for sending e-mail on the Internet or a network. Every mail server is capable of handling SMTP e-mail. Your agency can utilize this SMTP feature providing it has the proper mail server configuration. Contact the entity responsible for maintaining your mail server to confirm your SMTP settings:

1. If you are hosting your own e-mail server, contact your tech person that maintains or originally setup the server.

2. If you are using another party for your e-mail server, contact them by going to their website or calling their support line.

You’ll need to obtain the following information:

•

Your outgoing mailserver name (e.g., mail.domainname.com, smtp.domainname.com, or gomail.wspan.com).

•

E-mail account login and password if authentication to SMTP mail server is required.

 

•

TCP/IP Port used for e-mail. (Usually Port 25, but confirm this with your network administrator.)

•

ClientBase's Merge to E-mail via SMTP now supports SSL/TLS secure connections. There are four options available on the drop-down, None, TLS (if available), TLS, and SSL. The type of connection you select is determined by the type of security supported by your e-mail provider or Internet Service Provider (ISP). Please contact your ISP to find out if it supports SSL/TLS and which setting is correct for your e-mail account.

Step 1: To obtain a list of all user login records, log into the database as SYSDBA (System Database Administrator). Go to Utilities| Users Logins|User List.

(Please note: Users can access and setup their own login e-mail settings in addition to SYSDBA login by going to Utilities|My Login.

Step 2: Highlight each user login, click Modify|E-mail Settings. Change the default e-mail Client from MAPI to SMTP Client, from the drop-down list.

Step 3: Complete the User Login SMTP Settings obtained from the entity responsible for maintaining your mail server:

Default E-mail Client: Choose SMTP off the drop-down menu.

Default SMTP Mode: Offers different options for creating the body of the e-mail message. Select either Formatted Text, Document Templates, Raw HTML or Plain Text from the drop-down list. This setting determines the default Mode when using the Merge to E-mail feature, but can be changed when sending an e-mail.

From E-mail Address: Enter the e-mail address for the user login you are setting up. The address entered here appears as the from address for all e-mails sent from this user. If you wish to change this address frequently, do so at the e-mail level when you are sending it out.

In the From e-mail field, if you type joyce@abctravel.com and a message is sent, you see the From field saying joyce@abctravel.com. Some people prefer it to look like this instead, Joyce Smith. To have it appear like this, type Joyce Smith<joyce@abctravel.com> instead of just joyce@abctravel.com. You can also use your company name or department name; for example, Travel With the Stars<sales@TWTS.com> or Star Travel Marketing<sales@TWTS.com>.

Outgoing Mail Server: Enter the outgoing mail server name determined in your steps above. Examples: mail.domainname.com, smtp.domainname.com, or gomail.wspan.com).

Port: This setting defines the TCP/IP port which your e-mail application listens on for incoming mail. If left blank, this setting uses port 25 by default, which is the Internet standard for receiving SMTP mail. In some situations, you may have chosen to move the SMTP service to another port. Perhaps you already have a service on port 25, and you do not want the services to conflict over the port assignment. In such a case, enter the port ClientBase should use.

Authentication Type: This setting is used to authenticate your connection to the SMTP mail server by forcing a USER name and password.

Default: If the network login does not handle this authentication automatically, then set this field to Default.

None: If your agency is hosting your own e-mail server and the network login has been configured to handle the authentication to the SMTP Server automatically, set this field to None.

E-mail Account Login and Password: Use when Login Type is set to Default. Enter your e-mail account login and password here.

Secured Connections: If your Internet Service Provider (ISP) supports SSL/TLS secure connections, there are four options available on the drop down: None, TLS (if available), TLS, and SSL. The type of connection you select is determined by the type of security supported by your e-mail provider or Internet Service Provider (ISP). Please contact your ISP to find out which setting is correct for your e-mail account.

SSL: Use SSL for encrypting e-mail sent via SMTP.

TLS: Use TLS for encrypting e-mail sent via SMTP.

TLS, if available: Attempt to use TLS for encrypting e-mail sent via SMTP. If the server does not support TLS, SSL will be attempted instead.

None: You do not wish to encrypt your e-mail(s) or you SMTP server does not support SSL/TLS.

Note: TLS (Transport Layer Security) is quickly becoming the common standard for encrypting e-mails. SSL (Secure Sockets Layer) is the predecessor to TLS. There are very few differences between SSL and TLS. TLS's encryption is a little stronger, however both types are almost identical, with only minor technical differences in the way the e-mail is encrypted and transmitted.

The default port for encrypted connections is 465 and unencrypted is 25. ClientBase will automatically set the port to these defaults when choosing a Secure Connection setting. Note: Port 587 is also commonly used by many servers for encrypted connections.

Note: that in order for Secure Connection: TLS/SSL to work with SMTP E-mail settings in ClientBase Windows, you must install the 3rd party Win32 OpenSSL.

Track E-mail Broadcast Results

Tracking the success of your marketing efforts is important. The results from an e-mail broadcast may come in the form of an actual booking for the product you promoted, as an inquiry, or as a booking of another product. It is essential that to setup good tracking in the form of Mailers within ClientBase so that you can determine what vendors and marketing pieces get the most results.

Example: Agent Sharon sends out a Princess Alaska HTML e-mail broadcast to all clients interested in cruising and Alaska by doing a Level 2 Query from the client profile marketing tab. Before sending out the e-mail, Sharon clicks Mailer so the system can automatically create a record of this e-mail broadcast within each of the clients’ profiles. (After every e-mail blast for SMTP, you are also prompted "Create a Mailer?" before you leave the Merge to E-mail feature.)

When Sharon’s clients start replying to the mailer, Sharon keeps track of this by Querying up the Mailer and entering a Response Date. Because Sharon has created a Mailer, she can follow up in a few weeks to see what kind of results she’s gotten from the e-mail broadcast.

Reports on E-mail Marketing Results

The best part of doing e-mail marketing (outside of getting an actual booking) is using ClientBase to measure how successful the e-mail broadcasts are. It is essential that you create Mailers within ClientBase each time you sent out e-mails. Analyze your success by running queries or reports to see the bookings or activity of only those people who received a particular e-mail. To do this you need to know the name of the Mailer you created for a particular promotion.

E-mail Broadcast Booking Direct Results Report

To find how many bookings you received as a result of an e-mail broadcast, run the following Query and/or report that uses both the Res Cards and the invoices from travel history (in case your office does not use Res Cards for every reservation):

1) Open Level 3 Query and click Reset All.

2) Click Add Filter, and go to the Activities tab. In the Subject box, select Is Equal to. Then use the drop-down arrow to select the mailer promotion. Next, go to the ResCards tab and in Create Date, choose Is From, and type in a date using the date that you originally sent the e-mail broadcast, and in To, type today's current date. Press OK.

3) Click Add Filter and go to the Activities tab. In the Subject box, select Is Equal to. Then use the drop-down arrow to select the mailer promotion. Next, go to the Travel History tab and in the Invoice Issue Date, choose Is From. Type in a date using the date that you originally sent the e-mail broadcast, and in TO, type today's current date. Press OK.

4) Now you should have two filters in the Query window. Click Refresh.

All of the clients in this Query have received the promotional e-mail, and booked something from your office after they received the e-mail. You may also have some people who called and the agent created an Res Card that is Under Consideration. This means success! Now granted, it's possible that the client booked a different product other than what you were promoting. You'll need to open up their profiles and take a look at the Res Cards or travel history to see what they've booked.

E-mail Broadcast Multiple Vendor Report Using Res Cards

This is a comprehensive e-mail broadcast report based on the Res Cards alone, which you can use to judge if an e-mail broadcast increased the production for the vendor or possibly gave business to some of their competitors. (If you create reservations outside of the Res Cards, use Trams Back Office to pull vendor production reports.)

This multiple vendor report details all vendor production for a specific time period:

E-mail Broadcast Response Report

If you have been conscientiously entering the Response Date in the Mailer when clients are calling in about a marketing e-mail they received, you can print out a report to determine how many people have replied:

2) Go to the Activities tab and check Mailer. From the drop-down Completed/Responded menu, select Completed/Responded, and in the Date From Field, enter the date of the promotion, and in the To Field, enter today’s date.

All of the clients in this Query have responded to the promotional e-mail. If you have kept a record of how many e-mail you sent, you can get a percentage of how many people responded to the e-mail to determine if the e-mail marketing campaign was successful.

FAQ’s

Q: How can I cut and paste just an ordinary text from MS Word, the Internet, or any other program into my e-mail?

A: Highlight any text you want to copy and press <CTRL>C to copy. Then place your cursor in the e-mail screen, and press <CTRL>V to paste.

Q: How Can I Send Text Messages from Any E-mail Program

A: You can input a person’s cell phone number followed by the @ sign and a proper domain depending on the carrier (AT&T, Verizon, Sprint, etc.) as an e-mail address, and this indeed will send it as text message to them. To do this:

1. Enter it as an e-mail address entry.

2. Ask for the cell carrier.

3. Enter a description of SMS (or something like that) to help locate it.

Example: If my cell is 310-293-4444 and my carrier is AT&T, then e-mail me a text at 3102934444@txt.att.net.
