Air Configuration

Path: Air > Air Configuration Page

You can use the Air Configuration page to control flight-related fields and information displayed on the Home page, Select a flight page, and Review low fare options page. It also sets business rules for searching and pricing air itineraries.

To configure air

  1. Configure flight search options as follows:
    1. To enable the option to search by price, in the Enable the Search by price Search Option list, click Yes.
    2. Ignore the Display Speedy Air Search option? list as it is no longer supported.
    3. Select the Default Flight Search Results as follows:
      • To make Search by time the default search type on the Home page, click Standard Flight Selections.
      • To make Search by price the default search type on the Home page, click Instant Itineraries.
  2. Configure flight availability options as follows:
    1. If you want to limit flight options to only show flights with availability in the traveler's requested class of service, in the Limit Flight Choices to the user's requested service class list, click Yes. For example, users who request coach class do not see flights where coach is sold out, but first class seats are available. Otherwise, click No. The best practice is No.

    2. In the Sort flight search results by list, click the initial sort criteria used in the Search by time path.

      Note  Additional sorting options for the Search by price path and the Review low fare options page can be selected in Policy > Corporate Vendor Settings > Air & Car Setup.

    3. Indicate whether you want to Display all applicable fare classes.

      Notes   

      • This toggle works opposite to what the text implies. If you want to show all fare classes, click No. If not, click Yes.
      • If you want to display all fare classes, the traveler must have the skill level of expert. Skill level can be changed through the User Upload Tool or the user profile.
      • If this is set to No, the Traveler has the option of displaying all applicable fare classes (e.g. Class: F7 A7 P7 Y7 B7 H7 K7 M7 L7 W7 V7 G7 S7 N7 Q7 O7) by going to their profile and then the Flight preferences page and selecting the Show detailed seat availability information check box. Some frequent travelers like to see the classes available so they know if First is available for upgrade.

    4. Indicate whether you want to Display only the actual service class.

      If you want to see all service class and you set Display all applicable fare classes to No, then you would click No.

      If you want to see only the service class you requested, you set Display all applicable fare classes to set Display only the actual service class to Yes.

      Note  As the examples above show, this toggle works opposite to what the text implies.

    5. Ignore the Display non-discounted flight options list as it is no longer supported.
    6. Select how you want to Display Elapsed Flight Time as follows:
      • Do not display
      • Display for each flight segment – Time is shown for each segment only.
      • Display for each flight leg – Time is shown as a total for all segments (flight time + layovers).
  3. Configure pricing behavior of the system and the display of prices during the flight availability process:

    1. Indicate whether you want to Display prices on the availability pages for each flight choice as follows:

      When set to Yes, prices are displayed as price estimates when configured with the next list.

      If set to No, estimates are not displayed. (Best practice)

    2. If you selected Yes in the previous list, select how should prices be displayed as follows:

      • Display message to finish building itinerary – Prices are not displayed until a customer completes an itinerary. (Best practice) The traveler sees the message Finish Building Your Itinerary For Complete Price. Automatic if above toggle set to No.
      • Display price estimate – Toggle above must be set to Yes.
      • Display price estimate as Discounted – This displays the contracted discount fare estimate only.
      • Display price estimate as discounted and retail price – This displays both contracted and normal fare estimates.

      Note  Prices estimates are not recommended since they take the lowest round trip fares and divide by 2 and show that price as half of the one way fare. The traveler may choose a low outbound and return flight, but when combined, a much higher fare appears.

    3. Indicate whether you want to Use your negotiated airline fares?

      Note  If the negotiated fare code is not entered correctly or if the airline vendor has not loaded their rate to this code, the customer’s negotiated rates do not display or store. For Amadeus, the code must start with the letter “U.”

    4. In the Enter the special company code box, enter a special code for negotiated fares, often called a SNAP fare in Sabre, Private Fares in Apollo, or Secure Rates in Worldspan.

      Note  The system only supports negotiated rate codes in this box. If you add qualifiers, the booking cannot be created, and users get an error on the Trip review and checkout page.

    5. Indicate whether you want to Use negotiated fares for price estimates.

      If price estimates are enabled above, a price estimate appears, using your negotiated fares, for each flight leg that is based on round trip itineraries for the same carrier and same fare type. The best practice is No.

      Note  If enabling, price estimates may have been discounted twice. The system calculates this as low fare by taking the lowest nonrefundable fare in the market divided in half without taxes or surcharges. The system does not determine whether the user qualifies for the fare.

    6. Indicate whether you want to highlight negotiated airfares with that are returned from the GDS on the Review low fare options page if you do not have air contracts loaded into this system.

    7. Indicate whether you want to Always display negotiated airfares even if less expensive alternatives are available? If you click Yes, only your negotiated airfares are displayed, no lower priced options.

      Note  You may receive a call from Travelers that a carrier is not displaying. This may be because the carrier is not listed in MBP or is noted as a demoted carrier. Also, the carrier may not display because it does not contain a negotiated rate. This happens even if the site is set to request all airlines from the GDS.

    8. Indicate Which Search by price option would you like to enable? Options include

      • Fare Led Segments – Returns multiple best priced options displaying outbound and inbound flights on separate pages.
      • Fare Led Itineraries – Returns multiple best priced options displaying outbound and inbound flights together as itineraries.

      Note  The best practice is Fare Led Itineraries. Selecting Fare Led Segments can sometimes give the impression that the user has limited return flight choices.

  4. Configure fare led settings as follows:

    Note  Site Administrators can adjust these settings but should contact a GetThere representative if the settings are changed.

    1. For Sabre:

      1. In the Extend the search before and after the time the user selected list, click the search window you want the system to use.

        Note  GetThere sends three JR. commands to the Sabre system, each requesting 12 itineraries. Duplicates are removed, and the remaining results are merged and displayed to the traveler. Only one JR command can be customized.

      2. In the Indicate Fare Led search priorities boxes, type search priorities used in the JR. format sent to the Sabre system. The best practice is to leave these as the default (Price 1, Direct/Non-Stop 2, Carrier 3 and Time 4).

        Note  GetThere sends three JR. commands to the Sabre system, each requesting 12 itineraries. Duplicates are removed, and the remaining results are merged and displayed to the traveler. Only one JR command can be customized.

      3. Type the Number of connections included when performing a Fare Led Search. The logic behind this command is to return all flights including 1 connection. If it is a market where more than 1 connection is all that is available, then those flights are returned. Since we send 3 JR. commands, it is possible to see more connections in the matrix, even though we have requested only 1 connection.

      4. In the Sort the Fare Led Matrix by carrier Indicates whether the site administrator can set the sort order for flights. This is only available for fare led itineraries and fare led segments.

        If you click No, then set the sort order by entering a 1 or 2 in the Price and Number of stops boxes.

      5. In the Enable +/- days functionality list, indicate whether to allow users to search for lower fares on days surrounding their requested dates of travel. Set the maximum number of days to search on the outbound flight and the default outbound search. Then set the maximum number of days to search on the return flight and the default return search.

        Note  This only applies to the Coach class of service.

    2. For Worldspan:

      1. In the Extend the search before and after the time the user selected list, click the search window you want the system to use.

      2. Indicate Connections for Fare Led Search Options include

        • Display All Results – Display all results returned from the GDS.
        • Display direct/non-stop flights only – Display only direct and non-stop flights.
        • Exclude 2 connection flights – Do not display 2 connection flights.

        Note  GetThere sends 2 $P commands to the Worldspan system, each requesting 32 itineraries. Duplicates are removed and the remaining results merged and displayed to the traveler.

      3. Indicate whether to Search multiple airports. This uses the GDS co-terminal logic.

      4. In the Sort the Fare Led Matrix by carrier Indicates whether the site administrator can set the sort order for flights. This is only available for fare led itineraries and fare led segments.

        If you click No, then set the sort order by entering a 1 or 2 in the Price and Number of stops boxes.

    3. For Amadeus:

      1. In the Extend the search before and after the time the user selected list, click the search window you want the system to use.

      2. In the Indicate Fare Led search priorities boxes, type search priorities used in the JR. format. The best practice is to leave these as the default (Price 1, Direct/Non-Stop 2, Carrier 3 and Time 4).

      3. Type the Number of connections included when performing a Fare Led Search. The logic behind this command is to return all flights including 1 connection. If it is a market where more than 1 connection is all that is available, then those flights are returned. Since we send 3 JR. commands, it is possible to see more connections in the matrix, even though we have requested only 1 connection.

      4. In the Sort the Fare Led Matrix by carrier Indicates whether the site administrator can set the sort order for flights. This is only available for fare led itineraries and fare led segments.

        If you click No, then set the sort order by entering a 1 or 2 in the Price and Number of stops boxes.

    4. For Apollo/Galileo:

      1. In the Extend the search before and after the departure time the user selected list, click the search window you want the system to use when sending the form requests to Web-Services. This time range applies when a user selects times based on Leaves time on the Home page. GetThere defaults to 8 hours.

      2. In the Extend the search before and after the arrival time the user selected list, click the search window you want the system to use when sending the form requests to Web-Services. This time range applies when a user selects times based on Arrives time on the Home page. GetThere defaults to 8 hours.

      3. Indicate how to Display flight options for Fare Led Search Options include

        • Display All Results – Display all results returned from the GDS.
        • Display direct/non-stop flights only – Display only direct and non-stop flights.

        • Display max 1 connection – Display flights including one connections.

        • Display max 2 connection – Display flights including two connections.

        • Display max 3 connection - Display flights including three connections.

      4. Indicate whether to Search multiple airports. This uses the GDS co-terminal logic.

  5. Configure booking and ticketing options as follows:

    1. Indicate whether to Permit users to complete bookings of itineraries that cannot be priced by the GDS?. This is especially important for international travel.

      Note  This only applies when inventory is available.

    2. Indicate whether to Permit users to book a higher class of service if their requested class of service is unavailable?.

      If set to No, the system automatically searches for a flight in a lower class of service on behalf of a traveler who has requested a sold out higher cabin. If set to Yes, the system searches for a higher class of service.

    3. Select What is the minimum interval permitted between the flight time and the booking time?.

      If you want to restrict travelers from making bookings on the site at the last minute, you can set a time deadline by which bookings must be made prior to flight departure. If a traveler tries to book within the time window, they are given a warning message and are not able to proceed with the reservation.

      Note  This also affects travelers adding hotels and/or cars to an existing reservation if it contains air.

    4. Select How far in advance should tickets be issued for the user's purchased trip. The best practice is Last ticket date.

      Note  If the user purchases a 14 day advance fare, but this list is set to 24 hours instead of Last ticket date and the ticket is not issued until 24 hours in advance, the user does not receive the 14 day advance price.

    5. Enter the time difference between the host agency’s time zone and GetThere’s time zone (Pacific). This is required for correct enforcement of ticketing rules..

    6. Indicate whether to Enable Automated Electronic Upgrades. This is available on Sabre for American Airlines flights only. If the traveler has an eligible frequent flyer number and the itinerary is eligible for upgrade, you can allow the user to request upgrades to a higher class of service. After the user purchases a trip, the user can request upgrades on the Trips page.

    7. In the Prohibit Automatic Upgrade Requests within box, type the time before departure after which automated upgrade requests are prohibited.

  6. Configure passenger type configuration options to do the following:

    1. Select the Preferred passenger type.

      Note  Sabre customers: If the company has negotiated fares filed, in order for Sabre to return the desired price for your travelers’ trips, the preferred passenger type setting must be Other and the Other passenger type code must be NEG.

    2. In the Other passenger type code box, type the passenger type if the passenger type you want is not listed (NEG for Sabre if applicable).

      Note  For Amadeus Web Service sites, this box must contain a single passenger type code. No special characters can be used.

    3. In the Other fare type code box, type the other fare type code, if applicable.

      Note  For Amadeus Web Service sites, this box must contain a single passenger type code or fare type code. No special characters can be used.

  7. Configure alternate airports options for one or more airports as follows:

    Notes   

    • Alternate airport entries should be considered co-terminal by your host GDS.
    • The system may not show an availability for an alternate city if it is not compliant with your corporate travel policy.

    1. Enter the city code travelers search for that is close to other airports.

    2. Enter one or more airport codes that will trigger the return of availability for all airports servicing the city code, above.

  8. Configure other options as follows:

    1. If you are using the Amadeus GDS, indicate whether to Use NEGO fares?. Options include:

      • Published Pricing Only Modifier (P)
      • Yes, Use Agency Nego and Published Pricing Modifier (NP)

      • Yes, Use Unifares and Published Pricing Modifier (UP)

      Note  The best practice is to set this field to Yes, use Unifares and Published Pricing Modifier (UP) to allow non-corporate Unifares to be returned.

    2. For all GDS, configure the following:

      1. Indicate when you Display fare restriction information on the Review low fare options page. Options include:
        • Never
        • Once

        • Always (Best practice)

        Note  For fare rules to appear on the Trip review and checkout page, the Which sections of the fare rules should be displayed? box on the GT Administration page in the Air Configuration section must be configured by your GetThere representative.

      2. Select the maximum number of travelers per itinerary. You can select up to 9, but the best practice is 1.

        Notes   

        • More than 1 traveler requires additional configuration by your GetThere representative.
        • If you are using AAirpass, you must select 1.
  9. Click Save.
