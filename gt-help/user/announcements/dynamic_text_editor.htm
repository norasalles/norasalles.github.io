Dynamic Text Editor

Customized dynamic messages with the use of Velocity language are created in Site Administration.

Best practice is to create and format the message in the HTML Editor used for static messages, and then to cut it from the HTML Editor and paste it into the Dynamic Message box. All HTML statements must begin with <p> and end with </p>.

After you paste it into the Dynamic Message box, you can add the Velocity items in the Dynamic Message box.

Caution  While you can put Velocity statements, variables, references, or other items specific to Velocity into the HTML Editor, they must be moved to the Dynamic Message box. If they are left in the HTML Editor and then saved to the static Message box, the Velocity items appear as plain text.

Setting a Variable

VTL statements always begin with the # sign and are followed by the directive set.

Example:

#set

Velocity searches to find all # characters in which to start the creation of a VTL statement for display on the page.

Information following set, and enclosed in brackets ( ) are expressions, which are equations assigning some type of value to information that is not consistent, known as a variable.

Variables are listed at the start of the VTL statement and are designated as such with a $ sign, followed by the information to be captured. The value in quotations " " which follows a variable (s) is referred to as a directive, and as mentioned before, is used to do something. The variable and value must be separated by the equal = sign.

Example:

#set ($a = “GetThere”) <p>Thank you for booking with $a. </p>

In the above example, a variable and its value have been established. The template example above outputs “GetThere” any time there is ‘$a’ in the message.

A variable always leads with the $ symbol followed by the VTL Identifier. All alphabetic characters whether lower or upper case may be used as such. Other permitted characters accepted in these statements are as follows:

  • Numeric (0-9)
  • Hyphen (-)
  • Underscore (_)

Conditionals

If / ElseIf / Else

The #if directive captures true statements and allows messages to be included when, for example, a Hotel Search is conducted and the established condition is met.

Example:

#if($globals.hotelCity=="LAX")

<p>Airport Hotels within 10 miles of the airport must be booked, no exceptions!</p> #elseif($globals.hotelCity=="SAN")

<p>The La Jolla Inn should be booked and a rental car is permitted.</p>

#else

<p>All hotels require prior authorizations, except for SAN and LAX.</p>

#end

In this example, a condition has been set for LAX and another for SAN, while all other Hotel Search page results produce another message as defined in the above expression.

Once a VTL statement has been created, the user is permitted to modify the displayable text in the HTML Editor.

Or

When a message is to be displayed for two separate cities such as LGA or JFK, an or || condition is used.

Example:

#if($globals.hotelCity=="LGA" || $globals.hotelCity=="JFK")

<p>New York Hotels should be booked by the agency hotel desk 1-800-555-5555.</p>

#end

And

May be used to compare data, calling for each statement to be addressed and then compared for action.

#if($globals.hotelCity=="DFW" && $globals.hotelVendorExist("HI")

<p>DFW Holiday Inns offer Free WiFi and complimentary breakfast</p>

#end

Changing Message Content Based on Screen Size

You can now use special tags to designate portions of a message which should be suppressed on a given device, based on the screen size of that device. This allows you to show a different message based on the screen size. Three divisions of screen size are supported:

  • Small (sm) – mobile devices (smartphones) < 768 pixels wide
  • Medium (md) – tablets between 768 and 991 pixels wide
  • Large (lg) – laptop/desktop computers > 991 pixels wide

The special tags are <div> statements that reference CSS class names and “wrap around” message sections. Eight classes are provided:

  • msg-hide-sm-down – hide when device is < 768 pixels wide
  • msg-hide-md-down – hide when device is < 992 pixels wide
  • msg-hide-md-up – hide when device is > 991 pixels wide
  • msg-hide-lg-up – hide when device is > 767 pixels wide
  • msg-show-md-up – show when device is > 767 pixels wide
  • msg-show-lg-up – show when device is > 991 pixels wide
  • msg-show-md-down – show when device is < 992 pixels wide
  • msg-show-sm-down – show when device is < 768 pixels wide

These <div> tags can be used in standard messages using the HTML option or added to dynamic messages. When used, each class performs the following action:

The format of the entire special tag including message text is:

<div class="class-name-goes-here"> message text </div>

The following are examples:

<div class="msg-hide-sm-down"> This message is hidden on small mobile devices (smartphones), but will display on tablets or laptops/desktops. </div>

<div class="msg-hide-md-down"> This message is hidden on small mobile devices (smartphones) and tablets, but will display on laptops/desktops. </div>

<div class="msg-hide-lg-up"> This message is hidden on laptops/desktops but will display on tablets and small mobile devices (smartphones). </div>

<div class="msg-hide-md-up"> This message is hidden on tablets and laptops/desktops, but will display on small mobile devices (smartphones). </div>

Notes   

  • If you hide all the content of message for a display size, an empty modal or alert pop-up box appears.
  • These tags can only be used in the Detail Message section of carousel messages, not in the Summary Message section. Also, with carousel messages some portion of the message should be configured to display for each of the three sizes. Otherwise the carousel displays an empty message.
  • This feature is not available on the old Trip review and checkout page.
