No Hotel Reason Codes

Path: Policy > Corporate Travel Policy > No Reason Codes | No Hotel Reason Codes

You can use the No Hotel Reason Codes tab to add, edit, or delete reasons and reason codes for compliance or non-compliance when no hotel is booked. Travelers with non-compliant itineraries will see the reasons entered on this page and will be required to choose one prior to completing their booking. The reason they select can be included in the body of an e-mail that can be sent to the traveler's policy approver. Each reason can have an associated code that can be captured and formatted with a PNR editor or Modify PNR Editor string and documented in the PNR. For more information, see PNR Editor or Modify PNR Editor.

To configure no hotel reason codes

  1. In the Default No Car Reason Codes section, configure default reason codes as follows:

    1. In the Default Reason Code for Non-Compliance box, type the default reason code for noncompliance. You define the code.

      Note  These codes are not automatically entered into the traveler's PNR. You must contact your GetThere representative to enter a PNR edit string to move this information into the PNR.

    2. In the Default Reason Code for Compliance box, type the default reason code for compliance. You define the code.

      Note  These codes are not automatically entered into the traveler's PNR. You must contact your GetThere representative to enter a PNR edit string to move this information into the PNR.

  2. In the Custom No Car Reasons and Reason Codes section, define the reasons and reason codes for non-compliance that meet your travel program requirements for car reservations.

    Note  Use the New PNR and Change PNR editors to format the reason code selected by the traveler into the PNR reservation record.

    Do the following for each code you want to define:

    1. In the Code box, type the appropriate code regarding compliance. These are usually one to three letter combinations that can be inserted into PNR reservation records in the Host GDS.

      Note  These codes are not automatically entered into the traveler's PNR. You must contact your GetThere representative to enter a PNR edit string to move this information into the PNR.

    2. In the Reason that is displayed box, type the reason associated with this reason code. This information is displayed to travelers on the Out of Policy page.

    3. In the E-mail to list, click one of the following:

      • If e-mail messages should go to the traveler's travel authorizer, click Authorizer.
      • To override the traveler's travel authorizer and send e-mail messages to the site's default notification e-mail address, click Default e-mail.
