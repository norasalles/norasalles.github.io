Notification E-mails

Path: Policy > Corporate Travel Policy > Notification E-mails

You can use the Notification E-mails page to configure the e-mail sent to the Travel Authorizer/Approver when the traveler creates a non-policy compliant itinerary and you have set the system behavior to: Send e-mail and request policy exemption.

Note  The settings on this page override the Travel Authorizer e-mail address settings.

Configure Notification E-mail Settings

  1. In the Notification E-mail Header Message box, type the custom e-mail you want to send to the Travel Authorizer, or leave this box blank to use the following default message:

    This E-mail has been forwarded to you as a request to deviate from travel policy.

  2. Enter the “Reply-to” e-mail address that will receive the Travel Authorizer’s replies. This should be an e-mail account the Travel Agency monitors this e-mail account.

  3. Enter the “From” e-mail address that will appear in the notification e-mail (i.e., who sent the notification e-mail). The default is:

    GetThere Customer Service.

    Note  Some corporate networks do not allow an e-mail address outside the GetThere domain. Check with your network security administrator to ensure the e-mail address you enter in this field is valid for your intranet.

  4. If you have not assigned a Travel Authorizer or if the traveler selects a reason code that has a default e-mail address associated with it, enter an e-mail alias address that receives non-compliant e-mails in the Default notification e-mail address box.
  5. Click Save.
