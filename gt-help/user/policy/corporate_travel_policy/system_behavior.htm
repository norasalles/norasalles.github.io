System Behavior

Path: Policy > Corporate Travel Policy > System Behavior

Travel Policy System Behavior controls what the traveler is required to do if the system detects a non-compliant itinerary.

For more information, click one of the following:

Setup Tab

  1. Select the pages on which you want to hide or mark non-compliant options. These are the page(s) for which you want to either hide out of policy options or display an out of policy graphic for options that are not within policy.
  2. For each of the page types, select one of the following actions:

    To

    Select the

    Do nothing for out of policy options for these pages

    Do not display an out-of-policy icon for non-compliant options option button.

    Show for out of policy options for these pages

    Display an out-of-policy icon for non-compliant options option button.

    Hide all out of policy options for these pages

    Hide non-compliant options option button.

    If you choose the first or second option above, and the user selects an out of policy option, the user experiences one of the following behaviors as set in the travel policy:

    • Do Nothing – The system continues with the reservation with no impact on the user.
    • Force traveler to rebuild a compliant itinerary – The system informs the user that their itinerary is non-compliant and they are unable to complete their reservation. The user must start over and choose an itinerary that is within policy.
    • Allow traveler to select from a list of reasons – The system warns the user that their itinerary is non-compliant from a pre-established list of non-compliance reason codes before they complete their reservation.
    • Allow traveler to select from a list of travel authorizers – The system warns the user that their itinerary is non-compliant and they must enter a reason and select an authorized travel approver to review and approve their itinerary. The travel approver has no further action unless they wish to decline the travel.
    • Allow traveler to request policy exemption – The system warns the user that their itinerary is non-compliant. They must enter a reason for non-compliance and enter the purpose of the trip. The system passes this information by e-mail to the user's designated travel approver as specified on the Travel Policy Assignment page. The approver must forward their approval to the Host Travel Agency in order for the user's ticket to be issued.  

    If you choose the last option to hide out of policy options, this overrides any other behaviors set in the travel policy.

  3. If all itineraries, both compliant and non-compliant, need to be approved by the user’s Travel Approver before they can be ticketed, select the Require pre-trip authorization for all itineraries check box. A message is displayed to the user stating that the booking is not ticketed until approval has been received.

  4. If you do not want to use the default text, in the Pre-trip Authorization E-mail Message box, enter your own custom text to display to users when you choose Pre-trip Authorization.

    The following is the pre-trip authorization e-mail message default text:

    This E-mail has been forwarded to you as a request for permission to ticket a compliant travel itinerary. The following reservation will not be ticketed until your designated travel agency receives authorization from you.

    Please respond using the REPLY button on your E-mail editor. Your reply message will automatically be addressed to the proper destination. You must enter whether you are approving or not approving the itinerary, and select the SEND button on your E-mail editor. Your message will be sent to your travel agency and this itinerary will be addressed per your instruction.

  5. Click Save.

Messaging Tab

Note  This tab only applies to sites with Policy Administration Application (PAA) enabled.

  1. In the Custom Message for All Travel Types box, enter a custom warning message for noncompliant itineraries that appears on the Out of Policy page. This message appears for all noncompliant travel types (air, rail, hotel, or car), unless you enter a custom warning in one of the boxes below for the specific travel type. If this box and the box for the specific travel type are both blank, the default warning message appears.
  2. In the Custom Message for Air box, enter a custom warning message for noncompliant air bookings that appears on the Out of Policy page after selecting air travel. If this box remains blank, the message in the Custom Message for All Travel Types box appears, or if that box is also blank, the default warning message appears.
  3. In the Custom Message for Rail box, enter a custom warning message for noncompliant rail bookings that appears on the Out of Policy page after selecting rail travel. If this box remains blank, the message in the Custom Message for All Travel Types box appears, or if that box is also blank, the default warning message appears.
  4. In the Custom Message for Car box, enter a custom warning message for noncompliant car bookings that appears on the Out of Policy page after selecting a rental car. If this box remains blank, the message in the Custom Message for All Travel Types box appears, or if that box is also blank, the default warning message appears.
  5. In the Custom Message for Hotel box, enter a custom warning message for noncompliant hotel bookings that appears on the Out of Policy page after selecting a hotel. If this box remains blank, the message in the Custom Message for All Travel Types box appears, or if that box is also blank, the default warning message appears.
  6. In the Custom Message for the Trip Review & Check Out Page box, enter a custom warning message for noncompliant itineraries that appears on the Trip Review & Check Out page. This message appears for all noncompliant travel types (air, rail, hotel, or car). If this box is blank, the default warning message appears.
  7. Repeat steps 1-6 above for each language for which you want custom out of policy messages. If a language does not have a message, the system shows the message in the default language for the site.
  8. Click Save.
