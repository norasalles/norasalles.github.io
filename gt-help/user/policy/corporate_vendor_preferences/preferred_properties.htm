Preferred Properties

Path: Policy > Corporate Vendor Settings > Preferred Properties

You can use the Preferred Properties page to display a list of the Preferred Properties uploaded to the site via the Preferred Hotel Upload Tool. This is for display only. The list shows:

  • Primary Airport – The 3-letter primary airport code associated with the hotel as stored in the GetThere hotel database. This code may be different than the airport code uploaded using the preferred hotel database file.
  • Hotel Chain – The 2-letter hotel chain code from the Preferred Hotel Database Tool.
  • Property ID – The GDS code that is unique to that property.
  • Property Name – The full name of the hotel property. The booking tool displays this name to the user.
  • Preference type - Assign a numeric value for promoted tier level (1,2, etc.). Enter D to demote a property and display it last in the availability list. Enter X to excluded a property and remove it from availability.  The default is 1.
  • Rate 1:  Dates Effective and Rate 2:  Dates Effective – The rates and effective date information is from the rate1, begin_date1, end_date1 and rate2, begin_date2, and end_date2 fields that were loaded via the Preferred Hotel Database Tool.

Sort and Filter Preferred Properties

You can sort Preferred Properties by Airport (default sort), Hotel Chain, or Property Name. The secondary sorts are as follows:

After sorting, you can filter the results by selecting an available letter or number.

The system displays up to 50 preferred properties at one time. To view more, click the pagination links at the top of the chart.
