Maximum Number of Passengers - Air Passenger Count

Path: Policy > Maximum Number of Travelers > Air Passenger Count

You can use the Air Passenger Count page to enable this feature for the Main site. This works with Air Segment rules in Policy Administration where you have configured a rule based on the maximum number of travelers. After enabling, the system bases the decision on how you want the passenger count to be calculated on which sites are select on this page.

Notes   

  • A super user must enable the Maximum Number of Passengers functionality on the Policy > Super User page before you see this page or can configure how the count is determined.
  • You can only see and use the Air Passenger Count page on the Main site. It is not visible on a subsite.

Any bookings made in GetThere and all bookings that are acquired through PNR Acquire are used for the calculation. This includes the following:

  • All Flight segments, which includes passive air segments (GK, BK, HK, etc.), regardless of segment status.
  • PNRs on hold.
  • PNRs purchased.
  • Air segments/PNRs canceled. The system removes canceled segments from the passenger count when the booking information is updated every 24 hours.
  • Waitlist segments.

Note  Codeshare/operated by flights may not be counted accurately.

Caution  The existing manual Maximum Traveler functionality currently offered in the Air Segment rule in the Policy Administration Application is still available. If these rules are configured, along with the Automated Maximum Number of Traveler functionality, the Air Segment rules would both be evaluated and could result in duplicate and inconsistent evaluation violations on a flight.

The booking information data for flights is updated at 5 A.M.. and 7 P.M. Central Standard Time. If a company knew a flight had reached a maximum capacity within the last hour and needed to block it and the database table that GetThere uses to calculate passengers is not yet current, you could use an Air Segment rule to block bookings on the flight.

Travel agencies can only report which flights are closed using the provided PNR editor tags in the PNR when the Maximum Traveler Violation is invoked.

To configure Maximum Number of Passengers functionality for air

  1. If you want to limit the number of travelers based on traveler type, create custom fields (CFEs) for the different traveler types. For more information, see the Custom Field Edits, PNR Editor, and Profile Editor student guide.

    Notes   

    • The best practice is to use profile CFEs. If you use Trip review and checkout page (Itinerary review) CFEs, the traveler’s type is registered but is not checked for his/her booking because the entry of his/her type comes after the policy check.
    • The same custom fields for the traveler types must be copied to all sites you want to evaluate. You copy copy CFEs to existing sites using the Subsite Modificaiton Editor. For more information, see the Supersite Management student guide.

  2. In the Policy Administration Application, create a Maximum Number of Travelers rule. You can create a rule to check the maximum number of all travelers or the maximum number of specific traveler types on a flight.

    Note  The rule must be configured in the default policy for each subsite you want the system to evaluate for the maximum number of travelers.

  3. In Site Administration, on the Air Passenger Count page, select whether to Enable the evaluation for Maximum Number of Travelers rule.

    Notes   

    • If you want to report on flights deemed out of policy, you should configure any PNR Editor tags for the Maximum Traveler Violation.
    • You should ensure that you have any/all offline PNR’s going through PNR Acquire to ensure the most accurate data is being evaluated for calculation.
  4. Click one of the following options:

    To

    Do

    Evaluate the passenger count based on flights booked on each subsite separately

    (For example, if the maximum count was 10, site A has 6 passengers, and site B has 7 passengers, there would be no violation because each site is under 10 on a flight.)

    1. Click the Evaluate each flight within each subsite option.
    2. Click Save.

    Evaluate the passenger count based on flights booked on all subsites combined

    (For example, if the maximum count was 10, site A has 6 passengers, and site B has 4 passengers, if an additional passenger tried to book the same flight from any subsite, there would be a violation because both sites together would have more than 10 passengers on a flight.)

    1. Click the Evaluate each flight across all subsites option.
    2. Click Save.

    Evaluate the passenger count based on flights booked on one or more groups of subsites

    (For example, if the maximum count was 10, group A has 10 passengers, and group B has 4 passengers, if an additional passenger tried to book the same flight a subsite in group A, there would be a violation, but if an additional passenger tried to book the same flight from group B, there would be no violation.)

    1. Click the Evaluate each flight across a group of subsites option.
    2. Click Save.

    3. Click Add.

      The Modify Group dialog box appears.

    4. In the Group Name box, type the name for the group.

    5. In the Select sites to add to group list, select one or more sites for the group.

      Note  You can only add a subsite to one group.

    6. Click Save.
    7. Repeat steps c through f for each group you want to define. You can define up to 100 groups.
