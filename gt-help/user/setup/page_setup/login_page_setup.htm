Login Page Setup

Path: Setup > Pages > Login

This help topic shows you how to configure the following items on the Log In page:

  • Login boxes
  • Messaging

For more information, click one of the following:

Login Box

Path: Setup > Pages > Login | Login Box tab

You can configure the following items in the Log In page:

  • Custom Labels
  • Forgot Password
  • Create Account
  • Login Box

For more information, click one of the following:

  • Login Fields

    • The custom label feature allows you to replace the system default labels with customized text.

    • User Name – Appears on the Log In page, when creating a new account from the Log In page, and throughout Site Administration.
    • Password – Appears when creating a new account from the Log In page and throughout Site Administration.

    The settings of the main site apply to the Log In page. When set at the subsite level, the settings apply when creating a new user on a subsite and the Site Administration pages of the subsite.

  • Forgot Password

    • The Forgot Password feature allows users to retrieve forgotten passwords by clicking a link on the Log In page. When Allow users to recover a forgotten password is enabled, you can set up one of two methods for retrieving passwords:

    • Require the user to answer a question – When users click the Forgot password link, they are required to enter their user name. They are asked the question that they set up in their profile. If they answer the question correctly, they are prompted to change their password and are allowed access to the booking site. If users have not set up a question and answer in their profile, they cannot retrieve their password.
    • E-mail the password – When users click the Forgot password link, they are required to enter their user name. The booking site automatically e-mails a link the user can use to reset their password to the e-mail address saved in their profile. If users have not entered an e-mail address in their profile, they cannot  retrieve their password.

    If you have Single Sign On enabled, you cannot use this feature.

    If you do not allow users to retrieve their passwords, they must contact you or your internal support staff to retrieve or reset their password.

  • Create Account

    • The Create Account feature allows users to create a new user account from the Log In page and access your site. When you enable Allow nonregistered users to create a new account and access the site, the system displays a Create Account link on the Log In page. If you do not enable this feature, you must create accounts and set up the users' login ID/user names and passwords.

    If you do not Require new users to enter the subsite name and password key, when users click on the Create Account link, the system asks if they have a Site ID and Password Key. If they answer no, they continue with the Create Account process and are asked to enter a user name and password. Then they are asked to enter their name, address, telephone number, and e-mail address. After confirming the information, their account is created on the main site.

    If a user answers yes to Were you supplied with a Site ID and Password Key?, they are asked to enter the Site ID and Password Key and then continue with the Create Account process. After confirming the information, the system creates an account on the specified subsite.

    For added security, you can Require new users to enter the subsite name and password key. Enabling this feature prevents unauthorized access to your site. Users must enter the Site ID and Password Key.

    Do not enable this feature if you are using Single Sign On.

    You can also replace the system default labels with customized text.

    Note  This option applies to Supersites only.

    • Site ID – Appears when creating a new account from the Log In page and throughout Site Administration.
    • Password Key – Appears when creating a new account from the Log In page and throughout Site Administration.
  • Configure the Login Box
    1. If you want, enter different labels for User Name and Password.

    2. Select whether to Allow users to recover forgotten passwords, and then select the retrieval method.

      Note  If you have Single Sign On enabled, you cannot use this feature.

    3. Select whether you want to Allow nonregistered users to create a new account and access the site. If so, then select whether you want them to enter the subsite name and password key.

      Notes   

      • Recommendation: Do not allow users to create an account unless you also require them to enter the Site ID and Password Key. Most customer booking sites are proprietary. If your site is accessible by the public and you do not require a Site ID and Password key, any person with access can create a login and book travel on the site.
      • If you have Single Sign On enabled, do not enable this feature. The system automatically creates new user accounts when employees are added to the corporate intranet.
    4. If you want, enter different labels for Site ID and Password Key.
    5. Click Save.

Messaging

Path: Setup > Pages > Login | Messaging tab

You can display the following custom messages on the Log In page:

Type

Appears

Company Announcement

On the Log In page.

Login Box Announcement

On the Log In page above the log in boxes.

Bottom Message

On the bottom of the Log In page.

Note  The types of messages available on your system depends on the features you have enabled. By default these message areas are blank. There are no default messages. You can customize the messages for each language you have enabled.

To enter messages

  1. If applicable, click on the tab for appropriate language.
  2. Do one of the following:

    To add a message

    Do

    That is dependent on one or more conditions and/or requires JavaScript

    Enter information in the Top Dynamic Message box or Bottom Dynamic Message box. For more information, see the Dynamic Text Editor topic or the Velocity Student Guide.

    That always appear regardless of travel choices and does not require JavaScript

    Enter information in the Top Message box or Bottom Message box. For more information, see the Text Editor topic.

  3. To save your messages in Site Administration, at the bottom of the text editor, click Save. (You do not have to click Save each time you enter a message in a different language.)
