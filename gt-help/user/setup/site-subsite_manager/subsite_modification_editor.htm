Subsite Modification Editor

Path: Setup > Site/Subsite Manager > Subsite Modification Editor

You can use the Subsite Modification Editor page to modify Main Site/SubSites PNR Editor strings, Travel Policy, Market Based Preference, or Custom,Field Editor settings from the main site to the subsites. This allows you to automatically populate air, car, and hotel global preferences and move over city, country, or region preferences.

SubSites modify by:

  1. In the Enter the First Name type your first name.
  2. In the Enter the Last Name box, type your last name.

Existing Site and Subsite(S) to Be modified settings

  1. Select the existing site that will be copied from the list.
  2. Select the SubSites that will be modified. Select at least one site to copy to. The maximum number of subsites that can be selected is 200.
  3. Click Add.

Configuration settings

  1. In the Select configuration settings to be replaced list, select whether to copy Site Admin Settings or Travel Policy Settings from the list as follows:

    • Site Administration Settings - Configuration settings to be replaced
      • Select the PNR Editor check box. Enter the copy from line and to line numbers. The maximum number of strings that can be chosen per copy event is 5. Non-sequential strings should be separated with a comma; sequential strings are separated with a dash (1, 4, 7-9). The line numbers do not need to be the same, for instance, you can copy lines 1, 4 from the source to 3, 7 on the target sites.
      • Select the Modify PNR Editor check box. Enter the copy from line and to line numbers. The maximum number of strings that can be chosen per copy event is 5. Non-sequential strings should be separated with a comma; sequential strings are separated with a dash (1, 4, 7-9). The line numbers do not need to be the same, for instance, you can copy lines 1, 4 from the source to 3, 7 on the target sites.
      • Select a Corporate Vendor Market Preferences from the list. Only one preference can be copied at a time.
        • For the Regional Preferences, all Geographic regions are copied from the source site since the regional preferences do not work without them.
      • Select the Custom Field Editor Settings check box. All Custom Fields Edits from the source site to the target sites are copied.
      • Select the Profile Editor Lines check box. Enter the copy from line and to line numbers. The maximum number of strings that can be chosen per copy event is 5. Non-sequential strings should be separated with a comma; sequential strings are separated with a dash (1, 4, 7-9). The line numbers do not need to be the same, for instance, you can copy lines 1, 4 from the source to 3, 7 on the target sites.
      • Select the All Queue Master settings check box.
      • Select the All Hotel Reference Points check box.
      • Select the Custom Ticket Delivery configuration check box.
      • Select Copy from option check box and enter the Copy from option and to option numbers. The maximum options that can be copied is 1.
      • Select the Copy all rail options check box. The entire rail options section of the page is copied.
      • Select the Dynamic messaging (All Languages) check box. Select a dynamic messaging from the list. Only one messaging page can be copied at a time. This copies all content in the header/footer boxes of those pages in all languages if applicable, even if the same languages are not enabled on the target sites.
    • Travel Policy Settings - Travel Policy Settings to Replace
      • Select the Policies check box. This copies all policies from the source site to the target sites. If a policy with the same name already exists on the target sites, it will overwrite it. If no exact policy name match is found, it will add the policy as a new one.
      • Select the Regions check box. This copies all regions from the source site to the target sites. If a region with the same name already exists on the target sites, it will overwrite it. If no exact region name match is found, it will add the region as a new one.

      • Select the Locations check box. This copies all locations from the source site to the target sites. If a location with the same name already exists on the target sites, it will overwrite it. If no exact location name match is found, it will add the location as a new one.

        Notes   

        • Each option above is independent. You can copy only policies and nothing else. However, if you have a policy with region dependent rules and you do not also copy the regions, the policy will not work correctly because the regions will be missing. No special warning message is displayed to remind this.
        • An e-mail detailing requested and successful changes will be sent to the e-mail address in the online profile of the administrator making the changes.
  2. Click Save.
