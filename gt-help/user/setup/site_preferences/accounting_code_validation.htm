Accounting Code Validation

Path: Setup > Site Preferences > Accounting Code Validation

You can use Accounting Code of the users' cost center/company code at the subsite level. A user can then enter up to three cost centers at time of purchase and ensure the cost centers entered are valid. For more information, contact your GetThere representative.

  1. Select whether to Enable Accounting Code Validation on the Billing Information page.  If so, specify when the section should display.  
    1. Choose one of the following:

      If you select

      Then

      Always display the Cost Center section

      The Accounting Code Validation section displays for every booking.

      Only display the Cost Center section if a site card is selected

      The Accounting Code Validation section only displays if one of the site cards highlighted below is chosen from the Trip review and checkout page.

      1. Select whether to Allow users to allocate the cost between multiple account codes.
      2. In the Site Cards list click the site cards or direct bill account that require validation. To select multiple items, hold down the Ctrl key while you click other items that you want to select.
    2. Select Allow user to use default cost center from Profile page to display the traveler's cost center number entered in their profile as an option on the Trip review and checkout page.
  2. Select the maximum number of Accounting Codes users can enter. Click 1, 2, or 3.
  3. Select whether to Remove all spaces from accounting codes entered.
  4. Select whether to Change all letters to uppercase in accounting codes entered.
  5. Select whether to Enable Live Accounting Code Validation via Web Service.

  6. Select whether to Use the employee number from the user's profile when validating via Web Service.

    If you want the option to include the employee number in the validation test, a GetThere super user must select Accenture via REST as the service.

  7. In the Web Service Unavailable Message box, enter the message you want to display to users on the Trip review and checkout page if the Web service is unavailable.

  8. To change the Default Heading text for the Section Header Label, enter text in the Custom Heading box. This text displays on the Profile pages and Trip review and checkout page.

    Note  You can enter up to 30 characters, including special characters.

  9. To change the Default Label text for the Accounting Code Label, enter text in the Custom Label box. This text displays on the Profile pages and Trip review and checkout page.

    Note  You can enter up to 30 characters, including special characters.

  10. Select whether you want to Display a custom message at the top of the Accounting Code section on the Trip review and checkout page only. There is no default message.

    Note  You can enter up to 4,000 characters, including special characters.

  11. Click Save.
