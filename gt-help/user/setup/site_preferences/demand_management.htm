Demand Management

Path: Setup > Site Preferences > Demand Management

You can use Demand Management to capture the user’s purpose for travel and gather any additional data to support that reason.

You can enable Demand Management per site/subsite via a super user check box. Once enabled, Site Administrators can configure fields and create a “tree” structure for the purpose of trips section on the Home page. Demand Management supports multiple languages.

Purpose of Trip selections are stored in the database and can be used for future reporting needs. Selections are entered into the PNR using PNR Editor Strings.

For more information, click one of the following:

Configuring the display of Demand Management fields

You can use the Configuration tab to enable Demand Management on the site, as well as to add, remove, or reorder existing Demand Management fields as it will appear on the traveler's Home page.

Note  You do not have to enable Demand Management to add, reorder, or remove Demand Management fields.

For more information, click one of the following:

  • Enabling Demand Management for the site

    You must enable Demand Management for the site in order to show all the fields you configured on the traveler's Home page as follows:

    Note  You must create a primary dropdown before you can enable Demand Management for a site.

    1. Select the Enable Demand Management check box.
    2. Click Save.

  • Adding the primary dropdown

    Note  This procedure only applies when you have not added any fields to your Demand Management tree in the Display Configuration area.

    1. If this is the first field you are adding, click the Add Demand Management Dropdown link. The Add a Demand Management Dropdown dialog box appears.
    2. In the Select Demand Management Dropdown list, click the primary drop-down list for the Demand Management area, and click Add. The primary drop-down list and all the items configured in it appear in the Display Configuration area.
    3. If you want to add fields to an item in the primary drop-down list, do the following for each option:
      1. Click the Add Option link for that option. The Add Option dialog box appears.
      2. In the Option Type list, click the type of field you want to add to the option.

      3. In the Option list, click the existing field you want to add.

        Note  If you want to create a new field to add, click the Create a new field link. For more information, see Adding a New Demand Management Field.

      4. Click Add. The field appears under the item in the tree.
    4. Click Save.

  • Adding an option to the Demand Management tree

    Note  This procedure only applies when you have already added the primary dropdown to your Demand Management tree in the Display Configuration area.

    1. Do the following for each option you want to add:
      1. Click the Add Option link for that option. The Add Option dialog box appears.
      2. In the Option Type list, click the type of field you want to add to the option.

      3. In the Option list, click the existing field you want to add.

        Note  If you want to create a new field to add, click the Create a new field link. For more information, see Adding a New Demand Management Field.

      4. Click Add. The field appears under the item in the tree.
    2. Click Save.

  • Changing the order of Demand Management fields

    1. In the Display Configuration area, drag a field above or below another field.
    2. Click Save.

  • Removing a Demand Management field

    Notes   

    • When you remove a Demand Management field, it only removes it from the Demand Management area seen by the traveler. The field still exists and can be added later to the Display Configuration area.

    • You cannot use this procedure to remove an option from the primary dropdown. To remove an option from the primary dropdown, edit the primary dropdown and remove the option.

    1. For each field you want to remove, click the Remove link. A confirmation dialog box appears.
    2. To remove the field, click OK.
    3. Click Save.

Adding, Editing, and Deleting Demand Management fields

You can use the Field Setup tab add, edit, or delete a Demand Management field.

For more information, click one of the following:

  • Adding a new Demand Management field

    1. On the Demand Management page, click the Field Setup tab.
    2. Click Add. The Demand Management > Add New Field page appears.
    3. In the Database Field Name box, type a unique field name. This field name will be used to store user submitted values in the database. You can only use alphanumerical characters and underscores.

    4. If you want the field to be required, select the Make this field required for the traveler check box. When the field is required, the traveler will not be able to move beyond the Demand Management area to enter itinerary search criteria until they have entered the required Demand Management items.

      Note  This does not apply to message fields.

    5. In the Field Type list, click one of the following field types. Click the link for more information:

          3. Click Add. The item appears in the list to the right of the boxes, and an example of the list appears in the Example area.

      • Data Entry Box

        1. (optional) In the Acceptable Data Types list, click the types of data you want to accept in the data entry box. You can select any of the following data types:

          Note  To select multiple items, hold down the Ctrl key while you click other items that you want to select.

          • alpha - Alphabetical characters (either case) from A to Z, including spaces.
          • numeric - Numerical characters from 0 to 9. (cannot begin with a 0, use other instead)
          • special characters - All keyboard characters, except alphabetical and numerical.
          • date - A calendar date in either MM/DD/YYYY or DD/MM/YYYY format.
          • other - Characters that do not meet the above criteria but can be accepted in the field. When you select other, you must use the If "other," define box to define characters that can be used in the field. The following are examples:
            • If you want the field to contain limited numerical values, type, for example,0-5 to allow 0,1,2,3,4,5 only.
            • If you want the field to contain uppercase alphabetical characters, type A-Z.
            • If you want the field to contain lowercase alphabetical characters, type a-z.
            • If you want the field to contain a combination of all three with no spaces, type 0-5, A-Z, a-z.
        2. (optional) In the Minimum Value box, set the minimum number of characters that can be entered in the field, and select what type of characters, such as alphabetical and numerical, that are acceptable. For example, if your field must have at least 3 digits and a numerical value, type 000 to indicate the minimum number of characters as 3. If your field must have at least 3 characters and an alphabetical value, you would enter aaa or AAA depending on the case of letters you want to allow. You must always start with the lowest numerical or alphabetical character, starting with 0 or a or A. The Minimum Value box can contain a maximum of 64 characters.

        3. (optional) In the Maximum Value box, set the maximum number of characters that can be entered in the field, and select what type of characters, such as alphabetical and numerical, that are acceptable. For example, if your field must have at most 5 digits and a numerical value, type 99999 to indicate the maximum number of characters as 5. If your field must have at most 5 characters and an alphabetical value, you would enter zzzzz or ZZZZZ depending on the case of letters you want to allow. For a combination of lowercase and uppercase alphabetical characters, you can type AAA in the Minimum Value field and zzzzz in the Maximum Value field. For a combination of alphabetical and numerical characters, you can type 000 in the Minimum Value field and zzzzz in the Maximum Value field. Data is read in the order of the ASCII table, so uppercase comes before lowercase value, and numbers come before alphabetical characters. The Maximum Value box can contain a maximum of 64 characters.

          Notes   

          • If the traveler must enter a certain number of digits that may include leading zeros 00550, choose either alpha and numeric or other from the data type list. If the data type selected is numeric only, the system does not validate the leading zeros and allows the traveler to continue with less than the expected number of characters entered.
          • If you are allowing a space as a defined character it must not be the last definition entered. Likewise, if you are allowing a dash as a defined character, it must be the last item defined.
        4. (optional) In the Default Value box, type what you want to appear in the data entry box before the user types anything. If you leave this blank, the data entry box starts off blank.

          • other - Characters that do not meet the above criteria but can be accepted in the field. When you select other, you must use the If "other," define box to define characters that can be used in the field. The following are examples:
            • If you want the field to contain limited numerical values, type, for example,0-5 to allow 0,1,2,3,4,5 only.
            • If you want the field to contain uppercase alphabetical characters, type A-Z.
            • If you want the field to contain lowercase alphabetical characters, type a-z.
            • If you want the field to contain a combination of all three with no spaces, type 0-5, A-Z, a-z.
        2. (optional) In the Minimum Value box, set the minimum number of characters that can be entered in the field, and select what type of characters, such as alphabetical and numerical, that are acceptable. For example, if your field must have at least 3 digits and a numerical value, type 000 to indicate the minimum number of characters as 3. If your field must have at least 3 characters and an alphabetical value, you would enter aaa or AAA depending on the case of letters you want to allow. You must always start with the lowest numerical or alphabetical character, starting with 0 or a or A. The Minimum Value box can contain a maximum of 64 characters.

        3. (optional) In the Maximum Value box, set the maximum number of characters that can be entered in the field, and select what type of characters, such as alphabetical and numerical, that are acceptable. For example, if your field must have at most 5 digits and a numerical value, type 99999 to indicate the maximum number of characters as 5. If your field must have at most 5 characters and an alphabetical value, you would enter zzzzz or ZZZZZ depending on the case of letters you want to allow. For a combination of lowercase and uppercase alphabetical characters, you can type AAA in the Minimum Value field and zzzzz in the Maximum Value field. For a combination of alphabetical and numerical characters, you can type 000 in the Minimum Value field and zzzzz in the Maximum Value field. Data is read in the order of the ASCII table, so uppercase comes before lowercase value, and numbers come before alphabetical characters. The Maximum Value box can contain a maximum of 64 characters.

          Notes   

          • If the traveler must enter a certain number of digits that may include leading zeros 00550, choose either alpha and numeric or other from the data type list. If the data type selected is numeric only, the system does not validate the leading zeros and allows the traveler to continue with less than the expected number of characters entered.
          • If you are allowing a space as a defined character it must not be the last definition entered. Likewise, if you are allowing a dash as a defined character, it must be the last item defined.
        4. (optional) In the Default Value box, type what you want to appear in the password box before the user types anything. If you leave this blank, the password box starts off blank.

        5. Do the following in the language tabs:

          Notes   

          • The language tabs area defaults to the site's default language.
          • If any label field is translated, then all label fields must be translated for any given language. When translations do not exist for a configured language, the labels appear in the default language for the site.
          1. In the Field Label box, type the label that will appear to the traveler on the Home page.
          2. In the Instructional Text box, type and format any instructional text you have for the field.
          3. For each configured language, add the translated version of the field label and instructional text. Any language for which you do not add translated text appears in the site's default language.
        6. For each option you added, in the Option Labels area, click the tab for each language configured for the site, and enter the translation for the option.
    6. Click Save.

  • Deleting a Demand Management field

    Note  If you delete a field, the system removes the association of these fields on the Configuration tab, and you must update any associated PNR Editor strings.

    1. On the Demand Management page, click the Field Setup tab.
    2. In the Field Name column, select the check box to the left of each field name you want to delete.
    3. Click Delete. A confirmation dialog box appears.
    4. To confirm the deletion, click OK.The selected items disappear from the Field Setup tab.
