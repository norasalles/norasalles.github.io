Localization

Path: Setup > Site Preferences > Localization

You can use the Localization page to defines domestic travel and restricted countries.

To configure site localization settings

  1. Select your home country (i.e."domestic") for this site. The default setting is: USA.

  2. Select countries with restrictions (users will not be permitted to book travel to or from these countries):  This setting defines countries that users are not allowed to travel to or from. If a user attempts to make a reservation that includes an origin or destination in a country with travel restrictions, they receive a warning message indicating that they have chosen an illegal country and will need to submit a valid origin or destination. To create the list, hold down the Ctrl key while you click other items that you want to select.

  3. Click Save.

To configure the time settings

  1. Select whether to Use local time on your site.

    Best Practice: The default setting is disabled and the current time is set to PST.

    If you disable this setting, the system uses the server time as the site time, which is Pacific Standard Time (PST).

    If you enable this setting, the system uses the Greenwich Mean Time offset entered in the Time Difference area.

  2. In the What is the time difference between the local time of the site and Greenwich Mean Time (GMT) list, click the difference between GMT and local time. Time zones West of Greenwich, England use negative numbers and those to the East use positive numbers. For example, there is a minus 5-hour (-5h) difference between the U.S. East Coast (EST) and GMT. The current site time and server time are displayed in the fields below. The default setting is: 8h.
  3. Indicate Should the site's local time be updated for daylight savings. This setting controls whether the local time on the site observers daylight savings time or not. Some areas (e.g. Arizona) do not observer daylight savings. If you enable this setting, the system changes the time to reflect the Daylight Savings Time rules. If you disable this option, daylight savings is not observed. The default setting is enabled. If you enable this setting, enter When does daylight savings begin at the site's location and When does daylight saving's end at the site's location.

    Be sure to change these settings to apply to your state/country.

  4. Enter the time zone label that your site will display (e.g. PST, EST, etc.). Format: use 3 characters.
  5. Click Save.

To configure the currency settings

You can use the Currency section to control the definition, format, and other rules governing currency.

  1. Select the local currency that will be used on the site.
  2. Select Where should the ISO currency code display in relation to the number. This controls where the currency code (e.g. USD) appears with respect to the nominal currency amount (e.g. 1,000.00). For example, if this is set to Right, one thousand U.S. dollars will appear as 1,000 USD. If this is set to Left, it will appear as USD 1,000. Default would use the setting from the GDS. The default setting is Default.
  3. Select whether a currency symbol should be displayed with prices. For example, $, £.
  4. In the Override the currency type box, type the city or airport code for the ticketing site if you would like to override the currency type that is returned from the GDS pseudo-city that is hosting the site. Use this feature if you have a site in one country but the ticketing is done in another.
  5. Select Should the local currency be displayed for car and hotel prices based on the daily currency conversion that is sourced from the host GDS. If you select this check box, car and hotel prices display in the local currency for your site based on the daily currency conversion that is sourced from the host GDS. If this is disabled, prices display in the destination country currency.
  6. Click Save.

To configure miscellaneous settings

You can use the Miscellaneous section to define the default car transmission type and the site display standard.

  1. Select What is the default car transmission. The default setting is automatic.

  2. Select What display standard should other display items follow? This allows address fields to contain postal codes instead of zip codes. It also affects the display items where there is a difference between U.S. and international settings.

    Note  Canadian sites use the US standard displays settings.

  3. Click Save.
