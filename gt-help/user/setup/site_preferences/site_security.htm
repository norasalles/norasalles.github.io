Site Security

Path: Setup > Site Preferences > Security

Single Sign On (SSO) is an alternate authentication and log on process. It enables network users in a corporation to log onto the booking site without having to enter their user name and password again. If SSO is enabled, it passes information from your network servers that identify and validate the user. If these settings are enabled and not correctly matched with the corporation’s configuration, users will not be able to log on to the booking site.

To set up site security settings

  1. In the Browser Settings section, select whether to Permit users to encrypt and save their username and password in their browser cookie file. Login cookies enable the web browser to “remember” the user name and password. If you select this check box, an optional check box labeled Remember my Log-in Information is displayed on the Log In page. Checking this box permits the web browser to encrypt and save the user login information as a cookie. This information is then pre-populated on the Log In page the next time the user goes to it.
  2. In the Traditional Single Sign On section, set up Single Sign On as follows:
    1. Enter the Single Sign On Encryption Key. The Single Sign On encryption key is determined by the system administrator and cannot exceed 64 characters. You can enter any number of characters, but the key must be synchronized with the customer’s itn_isso.cfg configuration file. This key is case -sensitive and must exactly match.
    2. Enter the Number of Minutes for Time Deviation. For security reasons, a timestamp is included in the encrypted log on password. The timestamp must match the local time at GetThere in order for the user to gain access to the site. To account for Internet delays and/or slight differences in time between GetThere and the corporation, a time buffer/deviation is used. Enter a whole number to define this time buffer. If the encrypted timestamp and GetThere time deviate by more than the time entered, the user is denied access. The recommended number of minutes for time deviation is 5.
    3. Select the SSO type.

      Best Practice: You must select GetThere SSO.

    4. Enter the SSO URL string.
  3. In the Users section, select user settings as follows:
    1. Select whether to Allow new users to be added using Single Sign On. When you select this check box, new user accounts can be created using SSO.
    2. Select whether to Assume unknown users are new users. When a request is sent to Single Sign On, it includes a user_flag. This flag is set to either c for current or n for new. When you select this check box, Single Sign On creates a user even if the user_flag is set to c (current) and the system does not recognize them.
  4. Select whether to Allow updates to profiles if they are sent using Single Sign On. If you select this check box, profile updates can be done through Single Sign On.
  5. In the IP Mask and IP Address boxes, type the IP mask and IP addresses to which you want to restrict booking tool access.

    Please make sure the IP address of your host connection (server) is included in one of the IP address fields. If the IP address is not listed, you will not be permitted to re-access the site or the page to make a correction. Do not use this functionality if users access your intranet through proxy servers that employ dynamic IP addressing or if users connect to your intranet from home using ISP connections.

  6. Click Save.
