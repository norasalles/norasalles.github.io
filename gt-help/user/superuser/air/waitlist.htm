For more information, click one of the following:

Air Settings

Path: Air > SuperUser | Setup tab

The Settings tab allows you to set the localization, enable Air Canada Flight Pass, Air Connect settings, Waitlist, and Comparison Shopping.

To configure air settings

  1. In the Point of Sale list, click the country of ticketing for the site.
  2. Select whether to Enable access to Air Canada Flight Pass tool. This link appears in the More Booking & Planning Tools area on the Home page.
  3. Select whether to Enable Air Connect. This will enable the Air Connect page.

    Note  Low Fare Search must be enabled in conjunction with Air Connect functionality.

  4. Select whether to Show Southwest in Air Connect Sites list.

    Note  Before enabling Southwest Airlines via Air Connect you must have Executive Approval. The Southwest via Air Connect is only approved for limited accounts.

  5. Select whether to Enable the combining of Air Connect carriers and/or GDS carriers in a single itinerary on the Search by time path.

    Notes   

    • This supports mixing of Travelfusion and AgentWare carriers only. This is not available for BrazilXML carriers.
    • The mixing functionality only allows you to combine two different Air Connect carriers available through Travelfusion or AgentWare. It does not allow you to mix an Air Connect carrier with a GDS carrier or mix Travelfusion carriers with AgentWare carriers.
    • This does not apply to the Search by price path.
  6. Select whether to Enable Waitlists.

    Note  This feature is only available on Sabre and Abacus sites.

  7. Select whether to Enable Comparison Shopping. Comparison Shopping provides consolidated air availability results with multiple fares across cabins and fare types in a single display and provides the traveler the flexibility to create a simple round trip air itinerary or mix fare types.

    Note  Comparison Shopping is only available with the Sabre and/or Galileo GDS.

  8. Select whether you want to Enable Compare One Way Fares. Compare One Way Fares prices the selected itinerary as a round-trip fare and as one-way fares to obtain the best fare.

    Note  Enabling Compare One Way Fares is an optional function used in the Comparison Shopping feature. It is not available to be used independently.

    1. To apply a threshold amount when calculating the one ways fares to a round-trip fare, type an amount in the Threshold Amount box. The one-way fares must be less than the round-trip fares by at least the amount of the threshold in order for the system to show them. This accounts for agency and other fees that may be higher with one-way fares than they would be with a round-trip fare.

      Use whole numbers only (no decimals or dollar signs).

    2. If you added a threshold amount, in the Currency list click the currency for the threshold amount. The system converts the threshold/currency against the one way fares returned.
  9. Click Save.

SuperPNR Settings

Path: Air > SuperUser | SuperPNR

If you enable SuperPNR, the system integrates segment information obtained from multiple data sources into a single page. (This process does not affect the user experience.) You can configure up to 10 data source configurations.

For more information, click one of the following:

  • Enable SuperPNR

    1. Select Enable SuperPNR.

      Note  If you enable SuperPNR, you should add at least one SuperPNR configuration (see next procedure). If you disable SuperPNR, the system ignores any SuperPNR configurations listed on the page.

    2. Click Save.
  • Add a SuperPNR Configuration

    1. In the SuperPNR tab, click Add.

      • You can have up to 10 SuperPNR configurations per site. When you have 10, the Add button is disabled. If you need to add another one, you must first delete an existing configuration, then add a new configuration.
      • You can only have one Comparison Shopping configuration per subsite.

    2. Select the Secondary Data Source. The options are: Abacus (GTX1 only), Amadeus, Apollo, Galileo, Sabre, and Worldspan.

      Notes   

      • If the site usesGTX1 air shopping and the primary data source is Sabre and you have Enhanced Air enabled, you have an additional option: IntelliSell.
      • If the site usesGTX2 air shopping, you always have the IntelliSell option.
    3. Optional: If the site usesGTX1 air shopping and the secondary data source is not IntelliSell and you want to send an request to a specified TA Group to create the booking, do the following:
      1. Select Send follow-up requests. (If IntelliSell is the secondary data source, this option is disabled.)
      2. In the Follow-up Data Source list, click the data source where the booking functions should occur. (If IntelliSell is the secondary data source, this option is disabled.) The default option is: The Site's Primary Data Source. Other options include: Abacus, Amadeus, Apollo, Galileo, Sabre, and Worldspan.

      3. Type the Follow-up Data Source TA Group code where booking functions should occur. (If IntelliSell is the secondary data source, this option is disabled.) The default is blank.

        Use the format: <text>////<PCC>. The PCC format varies by GDS as follows:

        • Abacus: 4 alphanumeric characters
        • Amadeus: XXXYYZZZZ (XXX=3 letter code of office, YY=2 alphanumeric for the Corp., ZZZZ=4 character alphanumeric unique identifier)
        • Apollo: 3 or 4 alphanumeric characters
        • Galileo: 3 or 4 alphanumeric characters
        • Sabre: 4 alphanumeric characters
        • Worldspan: 3 alphanumeric characters
    4. In the TA Group box, type the code to use for the Data Source configuration. The TA Group tells the system where requests should be performed.

      Use the format: <text>////<PCC>. The PCC format varies by GDS as follows:

      • Abacus: 4 alphanumeric characters (GTX1 only)
      • Amadeus: XXXYYZZZZ (XXX=3 letter code of office, YY=2 alphanumeric for the Corp., ZZZZ=4 character alphanumeric unique identifier)
      • Apollo: 3 or 4 alphanumeric characters
      • Galileo: 3 or 4 alphanumeric characters
      • Sabre: 4 alphanumeric characters
      • Worldspan: 3 alphanumeric characters
    5. Optional: If the site usesGTX1 air shopping, do the following:
      1. Select whether you want to use Terminal Emulation or Structured Data for the Secondary Data Source Protocol. This option determines the method the system uses to connect to the secondary data source. Terminal Emulation is the most common method used.
      2. To tell the system when to request information from the secondary data source, select the When to Request Secondary-Source Information check box. Based on the option selected, that is the only time the system requests information from that data source. The options are:
        • Air Availability and Low Fare Search
        • Air Availability
        • Car Availability
        • Hotel Availability
        • Low Fare Search
        • Best Itineraries
        • When creating the PNR
        • When accessing the PNR
        • Car loc
    6. Optional: If the site usesGTX2 air shopping, select the Shopping Path option to tell the system which shopping path to request information from the secondary data source. Based on the option selected, that is the only time the system will request information from that data source. The options are:
      • Shop by price and time
      • Shop by time only
      • Shop by price only
    7. Optional: If the site usesGTX1 air shopping, do the following:

      1. If Sabre is the secondary data source and you selected Best Itineraries as the option for When to Request Secondary-Source Information, select the Shop by Price Option: choose Instant Itineraries (the default option) to return up to 12 best priced options or choose Fare Led to return multiple best priced options.

        If IntelliSell is the secondary data source is and you selected Best Itineraries as the option for When to Request Secondary-Source Information, select Fare Led as the Shop by Price Option to return multiple best priced options.

      2. To create a custom configuration of attributes used during the Low Fare Search, type the Low Fare Append String.
      3. To create a custom configuration of attributes used during Best Itinerary Search, type the Best Itinerary Append String.
    8. Optional: Do one of the following:

      If

      Then

      GTX1

      1. If you want to use a company pricing code when making requests to the data source, select Send company pricing code.
      2. Enter the Secondary-Source Company Code to use when making requests to the data source.

      GTX2

      Type the Secondary-Source Company Code to use when making requests to the data source.

    9. Select when to use the Secondary-Source Action. This setting tells the system if this data source configuration should replace the site’s primary GDS, or if it should supplement the site’s primary GDS. If you set the Secondary-Source Action to Replaces, then the site’s Primary GDS is not used when that data source configuration is used. If you set the Secondary-Source Action to Supplements, then the site’s Primary GDS is used in addition to the secondary data source being used. You can only have one SuperPNR configuration (out of the 10 that are allowed) set to Replaces. You can have multiple SuperPNR configurations set to Supplements.
    10. Select when to use Secondary-Source Use Status. This setting tells the system if this data source configuration should never be used, always be used, or only used for certain airlines.
      • If you set the status to Never, then that data source configuration is ignored. You can use the status of Never if you need to stop using a particular configuration for a period of time so you do not have to delete the configuration and then recreate it when it is needed again.
      • If you set the status to Always, the system checks that data source and uses it accordingly.
      • If you set the status to Only for the Airlines listed, then any time the system does use that configuration, it will only be for those airlines. If the Secondary-Source Use Status is set to Only for the Airlines listed, the Airlines text box is enabled. Enter the airline codes to tell the system to bias the request to that data source with that airline code or multiple airline codes. You can enter a maximum of 3 airline codes. If you are entering multiple airline codes, format as 2-letter airline codes followed by a comma.
    11. Optional: If you have Fare Led Settings, do one of the following:

      If

      Then

      Sabre (GTX1 and GTX2)

      Abacus (GTX1)

      1. Select the Search Time Range around User-Indicated Time to determine the time range to search according to the time the user selected.
      2. Determine the Fare Led Search Criteria Priority for price, direct/non-stop, time, and carrier.
      3. Select the maximum number of connections.

      Worldspan

      (GTX1 only)

      1. Select the Search Time Range around User-Indicated Time to determine the time range to search according to the time the user selected.
      2. Select whether to Display all results, Display direct/non-stop flights only, or Exclude 2 connection flights.
      3. Select whether to search for multiple airports.

      Apollo, Galileo

      (GTX2 only)

      1. Select the time range to search according to the time the user selected.
      2. Select whether to Display all results or to Display direct/non-stop flights only.
      3. Select whether to search for multiple airports.
    12. Optional: If you selected Amadeus in the Secondary Data Source list, do the following:
      1. If you want the validating carrier of the selected fare included in the format when the fare is stored, select the Store the carrier who validated the fare in the PNR. check box.
      2. In the Selling Point Override box, type an airport or city code to be used as the selling point in the entries to price and store fares.

      3. In the Ticketing Point Override box, type an airport or city code to be used as the ticketing point in the entries to price and store fares.

        Notes   

        • Once selections are made for a data source in a SuperPNR Setting, those selections remain in the data base for the data source until the SuperPNR setting is deleted. If the data source is changed, new selections are required. If the data source is then changed back to a previously selected data source in the same SuperPNR settings, the selections originally made for that data source are populated.
        • If SuperPNR is set up with a selling and/or ticketing point override and the office location (TA Group) is the same as the host GDS (TA Group Assignment), the selling and ticketing point overrides must be the same as in the Pricing and Stored Fare Options under Air Configuration. The system does not support the settings being different.
        • Sites must have Amadeus Web Services activated for the validating carrier, selling point override, and ticketing point override to fully work. The Shop by price and low fare searches for native Amadeus are not be programmed for these new options.
    13. To add the configuration and return to the SuperPNR tab, click Save. Or click Save & Add Another to add the configuration and leave the dialog box open so you can add anther configuration. (Click Cancel to return to the SuperPNR tab without saving your changes.)
  • Modify a SuperPNR Configuration
    1. In the SuperPNR column, select the link for the SuperPNR configurations you want to edit.
    2. Edit the SuperPNR information as needed.
    3. Click Save.
  • Delete a SuperPNR Configuration
    1. In the SuperPNR Configurations table, select the check boxes for the Super PNR configurations you want to delete.

    2. Click Delete.

      A confirmation dialog box appears.

    3. Click OK.
  • Delete all SuperPNR Configurations
    1. Select the check box in the header row.

    2. Click Delete.

      A confirmation dialog box appears.

    3. Click OK.
