Super User Hotel Settings

Path: Hotel > Super User

You can use the Hotel Super User page to control the source of hotel content, GDS or third-party content (BSI), to allow passive bookings and to enable and configure the use of non-GDS hotels through Hotel Connect.

To configure hotel settings

  1. Specify the hotel content type as follows:

    To

    Then

    Allow access to the Live Hotel Availability content which provides a better shopping experience

    Note  Whether or not you select this check box has no effect on Hotel Connect properties because Hotel Connect always returns live results.

    Select the Enable Live Hotel Availability check box. The default setting is not selected.  

    Specify the number of hotels to show at once on the Search Hotels page and when you click the Show more hotels link

    Note  If you have Live Hotel Availability enabled, this number also determines the number of LHA calls made per request. Therefore, if you are using a non-Sabre GDS, this may increase agency costs.

    Select the number in the Display hotels in increments of check box. The default setting is not selected.  

    Allow access to hotel content and to support Single Sign On

    Select Allow Site Administrators to enable Hotel Search (GDS and third-party content). The default setting is selected.

    Disable access to GDS and third-party hotel content

    Clear the Allow Site Administrators to enable Hotel Search (GDS and third-party content) check box.

  2. Configure Hotel Connect to allow access to non-GDS hotels as follows:
    1. To enable access to non-GDS hotels, select the Enable Hotel Connect check box.
    2. In the Hotel Connect aggregator list, click the aggregator you want to use to obtain non-GDS hotel content.
    3. In the Limit Hotel Connect results to list, click the maximum amount of non-GDS hotels you want to include in your search results. This limit affects system performance.
  3. Configure passive booking settings as follows:

    To

    Select

    Enable passive hotel segments for Non-preferred hotels and allow them to be booked passively if the hotel is sold out at the time of purchase

    Note  The PNR will contain, in addition to the Hotel Name, Rate and similar information, the 3 letter channel code followed by the channel confirmation number.

    The Create a passive booking segment in the PNR for a Non-Preferred Hotel Property if the available property cannot be booked when the user purchases an itinerary check box.

    Enable passive bookings for Preferred Hotel Properties that meet one or more of the following conditions:

    • The Preferred Property is sold out.
    • There are no room rates available equal to or less than the rate loaded in the preferred hotel database.
    • The hotel does not participate in the GDS.

    Note  The PNR will contain, in addition to the Hotel Name, Rate, and similar information, the 3-letter channel code followed by the channel confirmation number.

    The Create a passive booking segment in the PNR for Preferred Hotel Properties check box.

    To enable passive segments bookings for Non-preferred Hotel Connect properties

    Note  You can only select this check box if you select the Enable Hotel Connect check box.

    The Create a passive booking segment in the PNR for non-preferred Hotel Connect properties check box.

  4. Click Save.
