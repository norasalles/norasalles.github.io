Super User Rail Settings - GetThere Rail

Path: Rail > Super User > GetThere Rail

GetThere Rail is the newly branded name for Rail Connect. GetThere Rail is now integrated with the Sabre Rail Consolidator Platform for Amtrak. We currently have two tabs on the GetThere Rail page:

  • RailConnect – You can use this tab to configure the connection between GetThere and SilverRail Sabre Rail Consolidator Platform for Amtrak.
  • SabreRail – You can use this tab to configure the connection between GetThereand Sabre Rail Consolidator Platform for Amtrak or Trainline.

Note  You should only enable RailConnect (a.k.a. GetThere Rail) or SabreRail, not both.

For more information, click one of the following:

RailConnect Tab (a.k.a. GetThere Rail)

You can use the RailConnect to allow users to use GetThere Rail to shop, book, and cancel rail bookings for Amtrak, Deutsche Bahn, and Via Rail. You can compare corporate rates, rates for discount cards, and published rates across multiple classes of service for a single train option. You can compare the rules specific to the fares for each train option.

The options on this GetThere Rail Super User page apply to all subsites for this main site.

Note  You can only use RailConnect if you have enabled the new Home page and GetThere Express. Also, for Canadian sites, on the Localization page in the Miscellaneous area, in the What display standard should other display items follow? list, click US standard displays.

To configure GetThere Rail (a.k.a. RailConnect) settings

  1. To enable GetThere Rail settings for the main site, select the Enable RailConnect check box. You cannot enable GetThere Rail for a subsite unless it is enabled for the main site.
  2. In the Agency Code box, type the agency code for the GetThere Rail aggregator.
  3. In the Company Code box, type the company code for the GetThere Rail aggregator.
  4. In the Default Point of Sale Country list, click the country to use when the traveler profile does not have a country selected in the Personal Information page or when a request is being made for a guest traveler.
  5. To allow site administrators to manage GetThere Rail settings for their subsites, select the Allow Site Administrators to configure RailConnect check box.
  6. To enable passive bookings for GetThere Rail trips, select the Enable Passive Segments for RailConnect check box.

  7. In the Do not require nor allow users to enter the CVV number for the following channels list, click the channels for which you do not want to require users to enter a CVV number.

    Note  The rail channel must have already approved such bookings for that rail vendor and are able to fulfill the bookings without the CVV number.

  8. Click Save.

 

SabreRail Tab (Legacy Amtrak or Trainline)

You can use Sabre Railto shop, book, and cancel rail bookings for Amtrak through the legacy interface or Trainline. You can compare corporate rates, rates for discount cards, and published rates across multiple classes of service for a single train option. You can compare the rules specific to the fares for each train option.

Notes   

  • If you enable Sabre Railfor a subsite, it will only be available for that subsite. If you enable Sabre Rail for a supersite, it will only be available for the supersite, not all the subsites under the supersite.
  • For Canadian sites, on the Localization page in the Miscellaneous area, in the What display standard should other display items follow? list, click US standard displays.

To configure SabreRail settings

  1. To enable Sabre Rail settings for the main site, select the Enable SabreRail check box. You cannot enable Sabre Rail for a subsite unless it is enabled for the main site.

  2. In the TA Group box, type the TA group for getting Sabre Rail content. If the primary GDS is Sabre, then configure the same TA group under Sabre Rail. If the primary GDS is not Sabre, then configure a TA group that can be used to get Sabre Rail content. This TA group should be a Sabre TA Group.

    Note  There is a one to one mapping between PCC and Trainline Corporate Reference. Therefore, only one PCC can be used on a subsite to access Trainline. If a particular corporation uses multiple PCCs across different locations, they require a dedicated PCC and a dedicated subsite for each Corporate Reference.

  3. To allow site administrators to manage Sabre Rail settings for their subsites, select the Allow Site Administrators to configure SabreRail check box.
  4. To enable passive bookings for Sabre Rail trips, select the Enable Passive Segments for SabreRail check box.

  5. In the Do not require nor allow users to enter the CVV number for the following channels list, click the channels for which you do not want to require users to enter a CVV number.

    Note  The rail channel must have already approved such bookings for that rail vendor and are able to fulfill the bookings without the CVV number.

  6. Click Save.
