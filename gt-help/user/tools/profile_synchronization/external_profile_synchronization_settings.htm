External Profile Synchronization Settings

Path: Tools > Profile Synchronization > External Profile Synchronization Settings

For more information, click one of the following:

External Profile Synchronization Editor

You can use the External Profile Synchronization Editor area to manage synchronization settings between the GetThere profiles and the External Profile System (Sabre Profiles).

  1. If you want to add profiles you create or change in the GetThere profiles to the External Profile System (Sabre Profiles), select the Allow Synchronization from GetThere Profiles to External Profiles System check box.

    Note  This applies to changes made from this point forward.

  2. If you want to add profile changes you make in the External Profile System (Sabre Profiles) to the GetThere profiles, select the Allow Synchronization from External Profiles System to GetThere check box.

    Note  This applies to changes made from this point forward.

  3. If you want to synchronize all existing GetThere profile records to the External Profile System (Sabre Profiles), select the Perform Mass Profile Synchronization to Sabre Profiles check box. This creates new records and updates existing records in the External Profile System.

    Note  The system performs this one time after you select this check box. To perform this again, you must select this check box again.

  4. In the Default Association ID in External Profile System box, type the unique number defined in the External Profile System that links the traveler's profile to all the other profiles (Agency, Corporate, Operational, International, etc.), filters and formats in the External system (Sabre Profiles). GetThere sends this Association ID to Sabre Profiles with every newly-created GT profile that it synchronizes to Sabre Profiles.
  5. Do one of the following:

    If you want

    Do

    The system to use default settings of which fields to synchronize between GetThere and Sabre Profiles

    Select the Enable Default Mapping option button.

    This is the default setting.

    To allow a GT Super User (or Agency Super User for “string certified” sites) to choose which fields (or groups of fields) to synchronize back and forth between GetThere and Sabre Profiles

    Select the Enable Flexi Mapping option button.

    With Flexible Mapping, the mapping relationship between GetThere and Sabre Profiles fields is fixed (for example, GetThere’s Cost Center field can only map to Sabre Profile’s Cost Center field, not to a phone field in Sabre Profiles). Only the decision whether or not to synchronize updates of specific fields (or groups of fields) in GetThere to their pre-determined equivalent fields in Sabre Profiles (and vice-versa) can be made. You can make these specific field-by-field decisions on the Profile Field Synchronization page.

External GDS Profile Synchronization Log

The External GDS Profile Synchronization Log shows the different types of synchronization updates based on the initiating tool/module. For each category it shows how many profile records are:

  • In Queue – Waiting to be processed.
  • Processing – Currently in process.
  • Complete – Finished processing.
  • Pending – This is for users created with an external profile ID only and indicates we are waiting for an update from Sabre Profiles to come back before we allow anything else to use this profile.
  • On Hold – The sync is waiting to be processed because the external profile is being updated. When the update is complete, the system puts the profile in queue for syncing.
  • Failed – Failed to process.

Each time the page is refreshed, these numbers are recalculated.
