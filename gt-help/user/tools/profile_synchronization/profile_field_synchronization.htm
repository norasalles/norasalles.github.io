Profile Field Synchronization

Path: Tools > Profile Synchronization > Profile Field Synchronization

You can use this page to determine which fields or categories of fields are synchronized from GetThere to Sabre Profiles, and vice-versa, when a profile is updated in either system.

Notes   

  • Profile Field Synchronization only applies to sites using the Sabre GDS.
  • This page can only be edited by GT Super Users, or Agency Super Users on “string certified” sites. For more information, see Site Admin > Agency > Super User. The Allow Agency Super Users to add, modify and delete PNR Editor and GDS Profile Synchronization strings. check box must be selected. Agency Super Users on sites that are not string certified can only view the page.

If the Enable Flexi Mapping option button on the External Profile Synchronization Settings page is selected, you can use the Profile Field Synchronization page to determine which fields are synchronized.

Note  The Enable Flexi Mapping option button can be selected even when synchronization between the two systems is disabled. This allows Super Users to customize the field-by-field Flexible Mapping settings before beginning synchronization.

If the Enable Default Mapping option button on the External Profile Synchronization Settings page is selected, the site uses default settings for synchronization, and the settings cannot be changed on the Profile Field Synchronization page.

Some fields cannot be modified from the default values.

Some GetThere Profile fields (for example, User Password) cannot be synchronized to Sabre Profiles, and thus do not appear on this Flexible Mapping page.

Some categories (such as, Charge Cardsand Passports and Visas) can only be controlled at the aggregate level, meaning either all fields in the category are synchronized or no fields in the category are synchronized. For those categories, rows for the individual field names belonging to the category do not appear below the category header row.

GetThere custom fields (CFEs) are dynamically mapped to custom defined variables (CDVs) in Sabre Profiles. CDVs are created during the synchronization process if they do not already exist in Sabre Profiles.

Note  GetThere Profiles do not have dedicated fields for Mobile Phone and TSA’s Known Traveler Number, but Sabre Profiles does. If a GetThere CFE is created with the exact field name mobile_phone, then the system maps the field to Sabre Profiles standard field for Mobile Phone instead of dynamically creating a CDV with the same name. Similarly, if a GetThere CFE is created with the exact field name sfp_known_traveler_number, then the system maps the field to Sabre Profiles’ standard field for TSA Known Traveler Number.

To configure profile field synchronization

  1. Do one or more of the following:

    To

    Do

    Sync all changes from GetThere profiles to Sabre profiles

    In the table header row in the Sync from GT column, select the Sync All check box.

    Sync all changes for an entire category from GetThere profiles to Sabre profiles

    In the table header row in the Sync from GT column, select the Sync category check box.

    Sync all changes for an individual field from GetThere profiles to Sabre profiles

    In the cell in the Sync from GT column for the field you want to synchronize, select the Sync field check box.

    Sync all changes from Sabre profiles to GetThere profiles

    In the table header row in the Sync To GT column, select the Sync All check box.

    Sync all changes for an entire category from Sabre profiles to GetThere profiles

    In the table header row in the Sync To GT column, select the Sync category check box.

    Sync all changes for an individual field from Sabre profiles to GetThere profiles

    In the cell in the Sync To GT column for the field you want to synchronize, select the Sync field check box.

    Restore all default synchronization settings

    Click Restore Defaults.

  2. Click Save.
