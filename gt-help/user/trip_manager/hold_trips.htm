Hold Trips

Path: Trip Manager > Hold Trips

You can use the Hold Trips page to configure:

  • Setup options
  • Notification E-mails

The Hold Trips feature allows users to hold a reservation within the GDS for a pre-defined period of time without purchasing the trip. If they do not purchase the trip within the established time limit, the system automatically cancels the trip in the GDS.

For more information, click one of the following:

Setup Options for Hold Trip

Path: Trip Manager > Hold Trips | Setup tab

You can use the setup tab to enable and configure hold trips functionality.

To setup hold trip options

  1. To allow users to hold a reservation within the GDS, select the Enable Hold Trips check box.
  2. Select the Number of trips a user can hold (1, 5, 10, 15, or 20).

    Note  In the booking tool, when the user reaches the hold trip limit, the system disables the Hold Trip button.

  3. Select the Minimum number of hours before a flight that a trip can be placed on hold (2, 4, 6, 8, 16, 24, 48).

    Note  Best Practice: Use the same setting as the following item on the Air Configuration page in the Booking and Ticketing Options section: What is the minimum interval permitted between the flight time and the booking time?

  4. Select the Maximum number of hours to restrict GDS expiration date (24, 48, 72, None)
    • 24 , 48  or 72 - The booking is held up to 24, 48, or 72 hours, or by the ticket time limit provided by the GDS, whichever came first, before being canceled by the system.
    •  None - The booking is held up to the ticket time limit provided by the GDS before being canceled by the system.
  5. Select one of the following options:
    • To Allow users to purchase and hold trips
    • To limit their booking ability and Allow users to hold trips only.
  6. Determine how the system should respond if rates change between the time travelers place a trip on hold and when they purchase the trip.
    1. In the Type of price threshold for rate changes to Hold Trips list, select the unit of measure the system uses to calculate the price change limit:
      • None – The user can continue purchasing the trip without regard to a price increase.
      • Percentage – The system calculates the price threshold based on a percentage of the original price. For example, the user can purchase the trip if the new price is no more than 5% of the original price.
      • Dollar Amount – The system calculates the price threshold based on the dollar amount of the rate increase. For example, the user can purchase the trip if the new price is no more than $100 more than the original price.
    2. In the Amount box, type the percentage or dollar amount the system should use to determine the rate change threshold. Use whole numbers, no decimal points, dollar signs, or percentage signs.

      Note  Best Practice: Use "0". When travelers purchase the trip, the system displays a message notifying them the price has changed.

    3. Select the System response if rate changes occur when a user purchases a trip on hold and the rate change is greater than the price threshold as follows:
      • Display the Itinerary – The system displays the trip and allow users to continue

        Notes   

        • Best Practice: Use Display the itinerary.
        • When configuring this setting, keep in mind that if a user is forced to book a new itinerary because of a fare increase, the lower fare inventory may be sold out and the user may end up paying more.
      • Create new itinerary – The system advises users the price is no longer valid and they need to create a new reservation.
  7. Click Save.

E-mail Notifications

Path: Trip Manager > Hold Trips | E-mail Notifications tab

The E-mail Notifications tab allows you to send an e-mail to the travel approver and/or traveler when a trip is placed on hold.

To setup e-mail notifications

  1. Select whether to Send a confirmation e-mail when a trip is placed on hold.

    If you select this check box, select whether you want to send the e-mail to the travel approver and the traveler, the travel approver only, or the traveler only.

    Note  Best Practice: Use Travel Approver and user.

  2. Select whether to Display a header message in the confirmation e-mail.

    If you select this check box, specify the message as follows:

    • Use default message – If you select this option, the default message is:

      "This reservation has been placed on hold while you finalize your travel plans. Please be sure to purchase your itinerary before this trip is automatically canceled."

    • Use custom message – If you select this option, enter the message you want sent to new users. HTML is not supported in this field.

  3. Select whether to Send a warning e-mail prior to automatically canceling a trip on hold.

    If so, do the following:

    1. Select whether you want to send the e-mail to the travel approver and the traveler, the travel approver only, or to the traveler only.

      Notes   

      • Best Practice: Select Travel Approver and user and 4 hours.
      • If you select To travel approver and user or To user only and the booking is made by the traveler's travel arranger, the travel arranger also receives the e-mail.
    2. Select the Number of hours before auto-cancellation to send warning. (Default: 4 hours)
  4. Select whether to Send an e-mail when a trip on hold is automatically canceled. If you select this check box, do the following:
    1. Select whether you want to send the e-mail to the travel approver and the traveler, the travel approver only, or to the traveler only.

    2. Select whether to Display a header message in the auto-cancellation e-mail.

      If you select this check box, specify the message as follows:

      • Use default message – If you select this option, the default message is: This reservation was automatically canceled because it was not ticketed by the expiration date established during the booking process.
      • Use custom message – If you select this option, enter the message. HTML is not supported in this field.

      Note  Best Practice: Select Travel Approver and user

      Notes   

      • Best Practice: Select Travel Approver and user
      • If you select To travel approver and user or To user only and the booking is made by the traveler's travel arranger, the travel arranger also receives the e-mail.
  5. Type an E-mail address that will receive a notification when the system is unable to automatically cancel a held trip. An e-mail is sent to this address if the system is unable to cancel a trip on hold 1 hour past the hold trip expiration time.
  6. Click Save.
