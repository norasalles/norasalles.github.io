Super User Trip Manager

Path: Trip Manager > Super User

You can use the Trip Manager Super User page to enable and configure Ticket Manager functionality.

Note  Exchange, refund, and void functionality is supported for Sabre and Apollo sites/subsites when the Point of Sale is the United States.  Refund and void functionality is supported on Sabre sites/subsites when the Point of Sale is Canada.

The following criteria must be met in order for the ticket to be considered eligible for void, refund, or exchange:

  • Tickets were issued in and for travel within the United States, Puerto Rico, or U.S. Virgin Islands.
  • The most recent, active electronic ticket in the active PNR is processed for void, refund, or exchange.
  • The electronic ticketed itinerary must completely match the active PNR itinerary, including, but not limited to, booking code, fare basis/ticket designator, travel dates, origin/destination, carrier, and flight number.
  • Multiple tickets issued for the same itinerary are not supported.
  • The ticketed PNR must be active in the host GDS.
  • The ticket has not been previously refunded or voided.
  • The ticket has not been exchanged or is a result of a previous ticket exchange.
  • Refunds initiated by the user, including involuntary refunds, such as flights canceled due to mechanical problems, are not supported.
  • Exchanges initiated by the user, including involuntary exchanges, such as a flight schedule change, are not supported.
  • Itineraries are fully unflown, future date itineraries.
  • Itineraries are with a single carrier.
  • PNRs are for a single passenger.
  • Only one credit card form of payment was used, including Air Travel (TP) cards and corporate ghost cards.
  • Bookings are either one-way, round-trip, and multiple destination.

To configure online ticket exchange settings

  1. In the Enable ticket void, refund, and exchange list, click one of the following:
    • No – Disables Ticket Manager.
    • GTM Exchanges Only – Exchanges only are enabled.
    • GTM Refunds Only – Refunds only are enabled.
    • GTM Exchanges and GTM Refunds – Exchanges and refunds are enabled.
    • GTM Refunds and GTM Voids – Refunds and voids are enabled.
    • GTM Exchanges, GTM Refunds and GTM Voids – Exchanges, refunds, and voids are enabled

    Note  All other options shown are not supported.

  2. If you want the system to process the refund automatically, select GetThere processes refund fulfillment check box.
  3. If you want the system to attempt to process a void if a refund is requested within the Airlines Reporting Corporation (ARC) or Bank Settlement Plan (BSP) reporting period, select the Process a refund request as a void transaction when the request is within the ARC/BSP reporting period check box.
  4. Add Ticketing Locations based on the following GDS specifications:

    This box is required if:

    • The GetThere processes refund fulfillment check box is selected.
    • The Process a refund request as a void transaction when the request is within the ARC/BSP reporting period check box is selected.

    Hardcopy Printer/MIR Designation

    This box provides the printer designation where the settlement authorization, or refund exchange authorization (REA), is printed. The REA is required for ARC/BSP reconciliation of refund transactions.

    This box is required if:

    • The GetThere processes refund fulfillment check box is selected.

    This box provides the printer designation (usually invoice printer) where the settlement authorization, or refund exchange authorization (REA), is printed. The REA is required for ARC reconciliation of refund transactions.  

    This box is required if the GetThere processes refund fulfillment check box is selected.

    Additionally, the box is used to input a MIR device designation. The MIR device is the LNIATA that interfaces with the agency’s back-office accounting system.  Configuring a MIR device is optional, but the majority of corporate agencies interface to an accounting system for ARC purposes.

    To input both printer and MIR device designations, type the following:

    • HMLM(LNIATA)(device code)/(LNIATA)(device code)
    • Invoice printer device code is “DI”
    • MIR device code is “DA”

    Example:

    HMLM453D47DI/453D49DA

    TJR City Code

    This box is used to calculate the local time of the ticketing PCC when determining if the electronic ticket record issue date is within the ARC/BSP void window.  

    The city code listed in this box must match the ticketing PCC in the agency's TJR. To display the TJR details, enter: W/TA*PCC, for example, W/TA*G493. The TJR city code is listed in the "AA CITY" box.

    This box is required if the Process a refund request as a void transaction when the request is within the ARC/BSP reporting period check box is selected.

    This box is used to calculate the local time of the ticketing PCC when determining if the electronic ticket record issue date is within the ARC void window.  The city code listed in this box must the be city code of the ticketing pseudo.

    This box is required if the Process a refund request as a void transaction when the request is within the ARC/BSP reporting period check box is selected.

  5. To enable the suspended bookings on the main site and allow subsite administrators to enable suspended bookings, select the Enable Suspended Bookings check box.
