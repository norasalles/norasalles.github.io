Suspended Bookings

Path: Trip Manager > Suspended Bookings

You can use this page to enable Suspended Bookings and configure the Suspended Bookings Cancellation Queue. The Suspended Bookings feature allows site administrators to place bookings in a suspended state in the GetThere database and, based on site configuration, to not submit the booking for ticketing in the GDS. If a request to ticket the booking is not received within the established time limit, the system automatically cancels the bookings in the GDS.

To configure suspended bookings

  1. To allow site administrators to suspend a reservation in the GetThere database, select the Enable Suspended Bookings check box. Bookings can be made but are not submitted for ticketing, based on site configuration.
  2. Configure the Cancellation Queue as follows:
    1. In the Pseudo City Code for Bookings with Uncancellable Segment box, type the GDS PCC or Office ID to which the GDS PNRs are queued.
    2. In the Queue for Bookings with Uncancellable Segments box, type the queue number and Prefactory Code to be used in the GDS queue placement entry.
  3. Click Save.
