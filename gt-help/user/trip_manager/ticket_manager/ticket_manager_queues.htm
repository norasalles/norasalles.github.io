Ticket Manager – Queues

Path: Trip Manager > Ticket Manager > Queues

You can use the Queues page to configure queuing for the Online Ticket Exchange (OTE) process.

To configure queuing

  1. Enter the Pseudo City Code and Queue Number for successful exchanges.
  2. Enter the Pseudo City Code and Queue Number for failed exchanges.
  3. Enter the Pseudo City Code and Queue Number for successful refunds.
  4. Enter the Pseudo City Code and Queue Number for failed voids/refunds.
  5. Enter the Pseudo City Code and Queue Number for successful voids.

The following criteria must be met in order for the ticket to be considered eligible for void, refund, or exchange:

  • Tickets were issued in and for travel within the United States, Puerto Rico, or U.S. Virgin Islands.
  • The most recent, active electronic ticket in the active PNR is processed for void, refund, or exchange.
  • The electronic ticketed itinerary must completely match the active PNR itinerary, including, but not limited to, booking code, fare basis/ticket designator, travel dates, origin/destination, carrier, and flight number.
  • Multiple tickets issued for the same itinerary are not supported.
  • The ticketed PNR must be active in the host GDS.
  • The ticket has not been previously refunded or voided.
  • The ticket has not been exchanged or is a result of a previous ticket exchange.
  • Refunds initiated by the user, including involuntary refunds, such as flights canceled due to mechanical problems, are not supported.
  • Exchanges initiated by the user, including involuntary exchanges, such as a flight schedule change, are not supported.
  • Itineraries are fully unflown, future date itineraries.
  • Itineraries are with a single carrier.
  • PNRs are for a single passenger.
  • Only one credit card form of payment was used, including Air Travel (TP) cards and corporate ghost cards.
  • Bookings are either one-way, round-trip, and multiple destination.
