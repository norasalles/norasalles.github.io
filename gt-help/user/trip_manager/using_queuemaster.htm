Using QueueMaster

Path: Trip Manager > QueueMaster

QueueMaster sorts PNRs into specific queues based on how the PNR matches certain conditions. You can configure up to 12 queue options. The system applies the options in numerical order and sorts PNRs into the first three queues that match the criteria, unless you have selected to stop at this option. Then the PNRs go into the number of matching queues to that point.

For example, if Queue Option 1 has one sorting criteria and Queue Option 2 has two sorting criteria, a PNR record that matches all sorting criteria for both Option 1 and Option 2 is sorted into both queues unless Option 1 specifies to stop at that option.

If there are two or more Options with the same number of sorting criteria and a PNR matches ALL the criteria for those options, then the sorting Option with the lowest number takes precedence.

Be sure to test the Queue Options to ensure they are set up correctly.

Even though you can configure up to 12 queues, the system only queues to a maximum of 7 applicable queues per GDS in the following order:

  1. The first queue placement is the primary/default queue (if enabled) set at Agency > Agency Details > E-mail Address tab > Primary GDS Queue in the Primary GDS Pseudo City Code. If enabled, the system places all new, changed, and canceled PNRs in this queue.
  2. The second queue is a Ticket Manager success or failure queue (if applicable) set at Trip Manager > Ticket Manager > Queues.
  3. The system then queues to a maximum of 5 other applicable QueueMaster queues set on this QueueMaster page. If the default queue is not enabled and Ticket Manager does not apply, the system only queues to these 5 queues.

Notes   

  • If a PNR is queue placed using PNR Editor, then the PNR does not queue place based on QueueMaster settings regardless of whether the criteria has been met.
  • Queue placements are done in multiples of 3 and could be subject to simultaneous change responses.

To configure the queue options

  1. In the Drop the Online Booking Code in area, type the Pseudo City Code and Queue number that PNR records should be sorted into, provided they match the rest of the qualifying criteria you identify.

    Notes   

    • If you are using Sabre, add a prefatory instruction to each queue assignment (e.g. 190/11 where /11 is the prefatory instruction).
    • Test that all of the queues have been setup correctly. If the Pseudo City Code and Queue are incorrectly entered, PNR records will not be sorted correctly.
  2. Identify one or more of the following sorting criteria:
    If the itineraryEnter the following

    begins after

    If the itinerary meets the start date designated, the PNR is queued to this queue. Select the month and date from the list. This criterion works in conjunction with itinerary ends.

    and ends before

    If the itinerary meets the end date designated, the PNR is queued to this queue. Select the month and date from the list. This criterion works in conjunction with itinerary begins.

    contains a listed city

    Type the city codes that you want the system to match. Separate multiple city codes with a comma.

    contains a listed airline

    Type the two-character airline codes that you want the system to match. Separate multiple airline codes with a comma.

    contains a listed car vendor

    Type the two-character car vendor codes that you want the system to match. Separate multiple car codes with a comma.

    contains a listed hotel chain

    Type the two-character hotel chain codes that you want the system to match. Separate multiple hotel codes with a comma.

    contains a listed rail vendor

    Type the two-character rail vendor codes that you want the system to match. Separate multiple rail codes with a comma.

    Note  This is not applicable to SNCF reservations.

    is international (air)

    If the reservation takes the user outside of the country of origin, then select this check box to queue to the designated queue.

    contains special requests

    If the user enters text into the special request section, then select this check box to queue to the designated queue.

    is out of policy

    If the user is out of policy, then select this check box to queue to the designated queue.

    begins within

    If the travel begins within a specific timeframe, place on the designated queue. Select timeframe from the list.

    is to be delivered within

    If the reservation needs to be delivered to the traveler within a specific timeframe, queue to a designated queue. Select timeframe from the list.

    is booked when the office is

    If travel is booked when the office is open or closed, place on a designated queue. Choose open, closed, or leave blank.

    must be ticketed by

    If the reservation must be ticketed by a specific date measured in today, tomorrow, or the day after tomorrow, place on a designated queue. Choose today, tomorrow, day after tomorrow, or leave blank.

    contains a passive car or hotel segments

    If the reservation contains a passive segment, then select this check box to queue to the designated queue.

    contains a preferred hotel

    If the reservation contains a preferred hotel, then select this check box to queue to the designated queue.

    is a held trip

    If you have set up the Hold Trips functionality, then select this check box to queue to the designated queue.

    is a canceled itinerary

    If the reservation is a canceled itinerary, then select this check box to queue to the designated queue.

    contains Air Connect segments

    If the reservation has Auto or Mediated Air Connect booking segments, then select this check box to queue to the designated queue.

  3. In addition to the list of common conditional statements, at the bottom of each sorting option is an AND field. Use this field when the standard conditional statements are not sufficient to cover special situations. For example: a unique queue for the CEO of the company. The criterion entered must be in Quartermaster script (e.g. ?User,ta_policy="Corporate Officers"|) and may require assistance from support staff.  
  4. If you enable If selected, stop at this option, QueueMaster stops after it has finished comparing this sorting option to the reservation record. In this case, no sorting templates with a higher number (1-10) is used. If this option is disabled, QueueMaster continues comparing additional sorting options until it has found 3 templates that match the PNR, until it has gone through all of the sorting options, or until it comes to a sorting option with the if selected stop at this option enabled.
  5. Click Save.
