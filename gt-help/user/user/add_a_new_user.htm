Add a New User

Path: User > Add User

You can use the Add a New User page to create new user accounts. New users created here will not have to enter their passwords twice or enter a Site Name or Site Code on the Registration page.

To add a new user

  1. In the Personal Information section, enter optional personal information as needed. For more information, click here.

    Field Label

    Enter or select the...

    Title

    Personal title for the user.

    Name (first/middle/last)

    First, middle, and last name of the user.

    Employee Number

    Employee number for the user.

    Company

    Company name for the user.

    Division

    Division name for the user.

    Note  This information must be entered consistently as it will be used in GetThere DirectObserver reporting. Variations in division name result in data being sorted incorrectly.

    Cost Center

    Cost center for the user.

    Note  This information must be entered consistently as it will be used in GetThere DirectObserver reporting. Variations in division name result in data being sorted incorrectly.

    Department

    Department name or number for the user.

    Note  This information must be entered consistently as it will be used in GetThere DirectObserver reporting. Variations in division name result in data being sorted incorrectly.

    Business Phone

    Business phone number for the user.

    E-mail Address

    E-mail address for the user. The e-mail address is used for e-mail confirmation, password recovery, and other messages sent to users.

    Send a copy of travel reservations to

    Up to three e-mail addresses that receive booking confirmation and other travel reservation information for this user.

  2. In the User Information section, enter optional user information as needed. For more information, click here.

    Field Label

    Enter or select the...

    Subsite name

    Name of the subsite you want the user to be able to access.

    Note  Do not select a site if you want to create the user on the main site.

    Subsite login key

    Key (password) entered when the subsite is created.  If you have forgotten your subsite login key, please contact your GetThere representative.

  3. In the User Information section, enter the following required user information as follows:

    1. In the Enter the new user's username for login box, type a login name for the new user.  User names can be any combination of characters up to a maximum of 39 characters.

      Note  It is recommended that users not use an e-mail address for the user name.

    2. In the Enter the new user's password for login box, type a password for the new user, and verify by entering the password again. Passwords can be any combination of characters from a minimum of 5 to a maximum of 39 characters.

  4. In the User Information section, continue to enter optional user information as needed. For more information, click here.

    Field Label

    Enter or select the...

    Enter the new user's GDS profile name

    If your site uses profiles located in the Global Distribution System (GDS) and the profile name in the GDS is different from the login name, enter the new user’s profile name.

    CRS Location Code

    If the user’s profile, located on the GDS, is built off of a first level profile, enter the name of the first level profile.

    What is the new user's security level?

    New security level for the user from the drop down list. For a description of the security levels, click here.

    • Traveler – Can make reservations on the booking tool.
    • Travel Arranger – Person that can make reservations for another person or themselves.
    • Travel Approver – Person that can make reservations for themselves and approve reservations for others.
    • Agency Travel Counselor – Does not have Site Administration access but can make reservations for a user.
    • Profile Administrator – Can add users, keep Traveler profiles up-to-date.  Can book reservations for other users and guests.
    • Customer Help Desk – Can add users, keep profiles up-to-date, change passwords, and restore user accounts if they get locked out.
    • SubSite Administrator – Has administrative access to a specific SubSite only.
    • SuperSite Administrator – Has access to the Main site and any SubSites under the Main.
    • Agency Help Desk – Can add users, keep profiles up-to-date, change passwords, and restore user accounts if they get locked out.
    • Agency SuperUser – Has access to the Main site and any SubSites under the Main.  Can also view the Agency Tab.

    Force Immediate Password Change?

    Setting is always Yes and unable to change for sites not using Portal Registration and Single Sign-on.  All new users are forced to change their password upon their initial login.

    Note  This does not apply to sites using Portal Registration and Single Sign-On. If Portal Registration flag is enabled by GT Personnel, that box is no longer grayed out and the toggle can be changed to No.
  5. Click Save.
