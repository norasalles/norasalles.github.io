Automatic E-mails

Path: User > Automatic E-mails

You can use the Automatic E-mails page to configure the following:

  • Welcome Message
  • Password Confirmation message

For more information, click one of the following:

New User Welcome Message

Path: User > Automatic E-Mails | Welcome tab

When enabled and a Site Administrator creates a new account using the Add User page or when a user creates a new account from the Log In page, an e-mail welcome message is sent to new users.

Note  The e-mail is sent to the address provided during the creation of the new account.

To create a welcome message

  1. Select whether to Send a welcome e-mail to new users.
  2. In the From box, type the e-mail address that appears in the From field in the e-mail. The default address is support@getthere.com.

    Note  If you use an e-mail address in this field that does not include getthere.com, e-mails may be rejected by the corporate e-mail system. This message may be rejected if those e-mail servers determine that the domain shown and the transmitting message domain differ.

  3. In the Reply to box, type the e-mail address to which an e-mail will be sent when users reply to this message.
  4. In the Send errors to box, type the e-mail address to which a notification is sent if there is an e-mail delivery failure.
  5. Optional: In the Send copy to box, type the e-mail address to which you want to send a copy of each welcome message.

    Note  Leave this box blank unless this e-mail address needs to be informed every time a new user registers on the site or is manually added to the site by a Site Administrator.

  6. Select the welcome message as follows:

    • Use default message – If you select this option, the default message is:

      "Thank you for registering for your online corporate booking tool. If you need assistance when booking a trip, there is a Help link on every page."

    • Use custom message – If you select this option, type the message you want sent to new users. HTML is not supported in this box.
  7. Select Save.

Password Change Notification

Path: User > Automatic E-Mails | Password Confirmation tab

When enabled, a customized e-mail is sent to users when they successfully change their password.

Create a Password Confirmation

  1. Select whether to Send a confirmation e-mail when users change passwords.
  2. In the From box, type the e-mail address that appears in the From field in the e-mail. The default address is support@getthere.com.

    Note  If you use an e-mail address in this field that does not include getthere.com, e-mails may be rejected by the corporate e-mail system. This message may be rejected if those e-mail servers determine that the domain shown and the transmitting message domain differ.

  3. In the Reply to box, type the e-mail address to which an e-mail will be sent when users reply to this message.
  4. In the Send errors to box, type the e-mail address to which a notification is sent if there is an e-mail delivery failure.
  5. Optional: In the Send copy to box, type the e-mail address to which you want to send a copy of each welcome message.

    Note  Leave this box blank unless this e-mail address needs to be informed every time a new user registers on the site or is manually added to the site by a Site Administrator.

  6. Specify the password confirmation message as follows:

    • Use default message – If you select this option, the default message is:

      "This message confirms the password for your corporate travel booking tool was changed.

      If you did not change your password, please let us know by replying to this message.

      Thank you."

    • Use custom message – If you select this option, type the message you want sent to users when their password changes. HTML is not supported in this box.
  7. Select Save.
