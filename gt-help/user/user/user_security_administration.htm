User Security Administration

Path: User > User Security Administration

You can use the User Security Administration page to set security restrictions on login access to your site. You can add, edit, or delete a security level.

Add a Security Level

  1. Click the Add a new Security Level link.

    The User Security Editor page appears.

  2. In the Security level name box, type the name for the new security level.
  3. In the User level list, click the level of access allowed for this security level.

  4. In the User sessions expire after list, click the amount of time of inactivity after which a user session will expire for this security level.

    Note  The system no longer supports values higher than 15 minutes. If you select a value higher than 15 minutes, the system expires sessions after 15 minutes of inactivity.

  5. In the Log on attempts list, click the number of times a user can attempt to log on before they are locked out of the system for this security level.
  6. In the Password length requirement list, click the minimum length of the user's password for this security level.
  7. In the Password field digits list, click the number of numbers required to be a part of the password for this security level. If you do not want to require numbers to be a part of the password, click 0.
  8. In the Set password aging list, click the amount of months before users with this security level are required to change their password.
  9. In the Required password elements list, click one or more types of characters required to be in the password. To select multiple items, hold down the Ctrl key while you click other items that you want to select.
  10. In the Required username elements list, click one or more types of characters required to be in the user name. To select multiple items, hold down the Ctrl key while you click other items that you want to select.
  11. In the Password history list, click the when the user can reuse a password. If set to 3 for example, the user cannot use the same password until the 3rd password change.
  12. Ignore the Enable EasyReg list, as this function is no longer applicable.

  13. Click Save.

    The new security level appears on the User Security Administration page.

Edit a Security Level

  1. In the table, click the link in the Security Level Name column for the security level you want to modify.

    The User Security Editor page appears.

  2. Change the security level information as needed.
  3. Click Save.

Delete a Regional Market

  1. In the table, click the link in the Security Level Name column for the security level you want to delete.

    The User Security Editor page appears.

  2. Click Delete.

    The row for the security level disappears from the table on the User Security Administration page.
