Air Configuration Student Guide - INT

Objectives

Upon completion of this module, you will be able to:

  • Configure the air availability pages for your site.

  • Add Custom Airline Notes.

  • Configure AAirpass on a site.

 

 

Overview

This guide assists in configuring air on a site.  It reviews the various pages and toggles used in setting up the air availability pages and how they display to users.

This guide explains the Air menu in Site Administration.

 

Air Menu

 

Air menu consists of multiple toggles and settings. Some toggles may display only if enabled in the site such as Waitlist. The Air Configuration section comprises configuration for all the air settings under Air Configuration and branded fare configuration under Air Configuration More. All the other settings/toggles like Select Flight, Air Pages, Custom Airline Notes, Secondary Account Code and Air Extras for GDS allows to configure other aspects of air search.

Waitlist

The Waitlist feature allows a user to waitlist a flight segment if it is not available in the desired cabin at the time of booking. The waitlist is worked by the airline when cancellations are received on the desired flight. A prerequisites of configuration required by the GetThere configuration team before the below configurations appear on the Air tab in Site Administration. Raise a case or contact your site manager to complete the waitlist prerequisites.

Note

Waitlist functionality is only available for Sabre and Abacus GDSs.

The settings shown below control the itinerary options allowed in a booking that contains a waitlisted segment, the actions taken when a waitlist clearance is received from the airline through the GDS, and additional messaging.

Note

If this option is selected, then the flight options will be sorted by lowest to highest flight duration and this is calculated basis of first flight segment if connecting flight involved.

Example: if the traveler did the search for 9:00AM flights and there are 4 flights are available at 9:00am and out of 4 flights 3 are connecting and 1 is a direct flight. The duration for the 1 direct flight is 4hrs and the 3-connection flight total duration is 5hrs30min. However, the first flight in the connecting flight duration is 2hrs followed by a layover of 1.5hrs and then the next connection duration is 2hrs.

In this way, the connecting flight options will show first in the flight availability list instead of direct flight because the duration of first connection flight is less than the direct flight duration. This method of sorting will apply to every hour of flight options.

The below section controls the pricing behavior of the system and the display of prices during the flight availability process.

Option

Description

When set to Yes, prices are displayed as price estimates when configured with the next toggle.  If set to No, estimates are not displayed. (Best practice)

Note

This toggle only applicable for Non-branded GDS fare airlines only. This is not applicable for codeshare flights.

If yes, how should prices be displayed

  • Display message to finish building itinerary – If selected, only class of service name i.e. Economy, Business etc displayed. No other message displayed.

  • Display price estimate – Toggle above must be set to Yes and set this to Display price estimate.

  • Display price estimate as discounted and retail price – Discontinued

Prices estimates are not recommended since they take the lowest round trip fares and divide by 2 and show that price as half of the one-way fare.  The Traveler may choose a low outbound and return flight, but when combined, a much higher fare appears.

Select Yes if you have negotiated airfares so that they are shown.

If code is not entered correctly or if the airline vendor has not loaded their rate to this code, the customer’s negotiated rates do not display or store. For Amadeus, the code must start with the letter “U.”

This is a special code for negotiated fares often called a SNAP fare in Sabre, Private Fares in Apollo, or Secure Rates in Worldspan.

Note

The system only supports negotiated rate codes in this box. If you add qualifiers, the booking cannot be created, and users get an error on the Trip Review & Checkout page.

Use negotiated fares for price estimates

If price estimates enabled above, a price estimate displays, using your negotiated fares, for each flight leg that is based on round trip itineraries for the same carrier and same fare type.  Best practice is No.

Note

If enabling, price estimates may have been discounted twice.  The system calculates this as low fare by taking the lowest nonrefundable fare in the market divided in half without taxes or surcharges.  The system does not determine whether the user qualifies for the fare.

If you do not have air contracts loaded into this system, would you like to highlight negotiated airfares that are returned from the GDS on the Low Fare Search Results page?

It may possible that the GDS system has some auto price negotiated fare for the customer which doesn’t required any special company code/pricing code/snap code. Select this option as YES, to highlight these type of fare returned in air availability with negotiated fare icon.

Always display negotiated airfares

If you select Yes, only your negotiated airfares are displayed, no lower priced options.

Note

You may receive a call from Travelers that a carrier is not displaying.  This may be because the carrier is not listed in MBP or is noted as a demoted carrier.  Also, the carrier may not display because it does not contain a negotiated rate.  This happens even if the site is set to request all airlines from the GDS.

Note

Only applies at pricing. When shopping by schedule, it forces the selected option to a negotiated fare. When shopping by price, it reprices and possibly change the fare on the option selected by the traveler. The change in cost may not be shown until the user gets to the Checkout page. It does filter any fares out from the low fare options.

Which Shop by Price option would you like to enable

  • Fare Led Segments – Returns multiple best priced options displaying outbound and inbound flights on separate pages.

  • Fare Led Itineraries – Returns multiple best priced options displaying outbound and inbound flights together as itineraries.

Note

Best Practice is Fare Led Itineraries. Selecting Fare Led Segments can sometimes give the impression that the user has limited return flight choices.

Alternate Airport

Before being with the setup you must be aware of alternate airport concept as general. The GetThere system identifies the airports requested for arrival/departure by the user as primary airport and any other airport options returned in the fight option will be identify as alternate airport.

So, if the user is searching for JFK as arrival airport then the flight options with JFK as arrival airport will be primary airport and the flights options with EWR and/or LGA as arrival airport will be alternate airport.

Similarly, if the user is searching for EWR as arrival airport then the flight options with EWR as arrival airport will be primary airport and the flights options with JFK and/or LGA as arrival airport will be alternate airport.

The configuration of alternate airport is not the same as it is explained in the configuration section. So, follow below guidelines to configure alternate airport properly.

Alternate airport configuration allows you to setup a default airport as per your choice and make other airport options restricted to display as itinerary option.

The airport code entered in the city code box will be the default airport and the airport codes entered in the airport codes box will be treated as alternate airport and will be restricted to display as itinerary option.

As per the above screenshot, JFK has been setup as default airport and EWR and LGA are the restricted alternate airport. In this way, if a user is searching for EWR and/or LGA as departure/arrival airport then the system will bring flight options with JFK only and show JFK with alternate airport icon.

 

If alternate airport not configured:

If the user search for LAX to LGW, then the flight options which has LGW as destination, identified as primary airport and if there are flight options with other airport available within the departure time window, then it will be considered as alternate airport i.e. LHR.

Custom Airline Notes

Custom Airline Notes creates a hyperlink (and if selected, an icon) that takes the user to detailed information about specific flights on the Select a Flight page. The hyperlink is applied to all flights that match the criteria specified and can include a custom message about a discount, negotiated amenity, other promotion, or convey messages to users concerning reasons to avoid certain flights. For example, you can configure Custom Airline Notes to warn travelers to avoid a specific airline that has repeated safety violations.

Note

Custom Airline Notes do not create discount fares. They only apply a hyperlink to flights that match the specified criteria.

The following are examples of using wild cards:

  1. To find any fare basis code that ends in the number "9" (applies to any number 0-9), enter the following format in the Fare Basis Code(s) box: regex(.+9$)

  2. To find any fare basis code that ends in the letter "R" (apply to any letter in the alphabet), enter the following format in the Fare Basis Code(s) box: regex(.+R$)

  3. To find any fare basis code that ends in the special character "@" (apply to any special character), enter the following format in the Fare Basis Code(s) box: regex(.+@$)

  4. To find any fare basis code that begins with number, letter or special character, enter the following format in the Fare Basis Code(s) box: regex(^[9|R|@].+)

    1. This statement means FB starts with ‘9’ or ‘R’ or ‘@’

    2. Custom Airline Note would apply to a FB that was a R21NR

    3. Custom Airline Note would NOT apply to a FB that was F9R@

  5. To find any fare basis code that BOTH begins with the number, letter or special character specified AND ends with the number, letter or special character specified, enter the following format in the Fare Basis Code(s) box: regex(^[9|R|@].+[9|R|@]$).

    1. This statement means FB starts with ‘9’ or ‘R’ or ‘@’ AND ends with ‘9’ or ‘R’ or ‘@. The “.+” in the middle of statement means there can be any characters in between.

    2. Custom Airline Note would apply to a FB that was a R21NR

    3. Custom Airline Note would NOT apply to a FB that was F9R@

  6. To find any fare basis code that includes the number, letter or special character specified, enter the following format in the Fare Basis Code(s) box: regex(.+[9|R|@].+)

    1. This statement means FB starts with ‘9’ or ‘R’ or ‘@’ AND ends with ‘9’ or ‘R’ or ‘@. The “.+” in the middle of statement means there can be any characters in between.

  7. To find any fare basis code that includes the number, letter or special character specified, enter the following format in the Fare Basis Code(s) box: regex(.+[9|R|@].+)

Note

“regex” should always be in lowercase and is used as an identifier for the rest of the expression inside the “()” as the code pattern.
