Trams Back Office 4.10.00

New Features and Enhancements - Released April, 2021

 

Trams Back Office 4.10.00 Beta New Features and Enhancements

Enhancements made between Trams Back Office 4.10.00.002 and 4.10.00.015

 

Branding

  • Updated initial set of branding to reflect Tres Technologies ownership of Trams and ClientBase products effective 4/1/2021.

 

CSL

  • Updated how Trams Back Office matches hotel profile IDs when interfacing HHX type bookings created by the Sabre CSL hotel booking solution.
  • Updated how Trams Back Office creates the Vendor profile interface ID if no profile is found:  If no user supplied code is provided, the Hotel Chain Code and Property Code will be used to create the Interface ID.  If there is no Property code in the interface record, the Sabre Global ID will be used to create the vendor interface ID.
  • Added the option of configuring a UDID in Utilities/Set Up/Global Defaults/Interface Mailmerge Prompt to indicate the CSL aggregator for reporting and analytics.

 

Bank Masking

  • Updated bank accounts stored in Profiles to be masked in the format XXXX-NNNN
  • Updated bank account number masking for ACH Payments
  • Added a prompt to remind user to view full/unmasked credit card or bank account number before updating or correcting the number.
  • Added a “New” button to streamline data entry for new credit card and bank account numbers in profiles.
  • Updated the User Permission description label to include account numbers as well as credit cards.

 

Note:

Please note that after the new Trams Back Office release has been installed, all bank account numbers entered in profiles and payments will be correctly masked. To mask previously entered bank account details, you will need to run the DBUp. If you do not capture bank account details in payments or profiles, you do not need to run the DBUp.

If your Trams Back Office syncs to any ClientBase for Windows copy databases, please install this Trams update, run the DBUp on your Trams Back Office database and then run the DBUp on your ClientBase for Windows databases to ensure the bank account details in your profiles and payments are correctly masked. (If your Trams and ClientBase for Windows do not sync because they are both connected to the one database in an office, then you do not need to run the DBUp again).
